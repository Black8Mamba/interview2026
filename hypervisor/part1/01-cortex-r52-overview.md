# 第 1 章：Cortex-R52 架构概述

## 1.1 处理器定位与应用场景

Cortex-R52 是 ARM 公司面向实时应用的高性能处理器，属于 ARMv8-R 架构体系。该处理器主要应用于安全关键（Safety-Critical）和实时关键（Real-Time Critical）领域，包括：

- **汽车电子**：自动驾驶域控制器、动力总成控制
- **工业控制**：工业自动化、机器人控制
- **医疗设备**：医疗影像、诊断设备
- **航空航天**：飞行控制系统

Stellar SR6P3c4 是瑞萨电子（Renesas）推出的车规级 MCU，集成 Cortex-R52 核心，专为汽车域控制器设计，支持 ASIL-D 功能安全等级。

## 1.2 ARMv8-R 架构特性

ARMv8-R 是 ARM 首次在 Cortex-R 系列中采用 64 位架构，支持 AArch64（64位）和 AArch32（32位）两种执行状态。

### 主要特性

- **64位架构**：支持 ARM 指令集和 Thumb-2 指令集
- **浮点单元**：支持 ARMv8 NEON SIMD 扩展
- **特权级别**：具备 Exception Level (EL) 区分，支持虚拟化

### 与 Cortex-R5 的对比

| 特性 | Cortex-R5 | Cortex-R52 |
|------|-----------|------------|
| 架构 | ARMv7-R | ARMv8-R |
| 位宽 | 32位 | 64位 |
| 虚拟化 | 无 | 支持 EL2 |
| 内存保护 | MPU | MPU (增强) |
| 最高安全等级 | ASIL-B | ASIL-D |

## 1.3 多核支持与锁步模式

Cortex-R52 支持多种运行模式，以满足不同应用需求：

### 1.3.1 多核模式（Split-Lock）

- **分离模式（Split）**：多个核心独立运行，各自执行不同任务
- **锁步模式（Lockstep）**：两个核心执行完全相同的指令，提供硬件级冗余

### 1.3.2 空间隔离

Cortex-R52 支持处理器核心间的空间隔离，每个核心可以运行不同的软件栈：

- 每个核心有独立的 MPU 配置
- 支持硬件强制隔离
- 适用于功能安全要求

### 1.3.3 集群架构

Stellar SR6P3c4 采用多集群架构：

- 最多 3 个集群（Cluster）
- 每个集群最多 2 个核心
- 每个核心独立运行或配置为锁步模式

## 1.4 内存保护单元（MPU）

Cortex-R52 集成了可配置的内存保护单元（MPU），是实现虚拟化隔离的关键组件。

### 1.4.1 MPU 特性

- **保护区数量**：最多支持 16 个保护区
- **区域大小**：从 4KB 到 4GB 可配置
- **访问权限**：读、写、执行权限控制
- **内存属性**：缓存属性、共享属性配置

### 1.4.2 虚拟化中的 MPU 用途

在 Hypervisor 环境中，MPU 用于：

- 隔离 Hypervisor 与 VM 的内存空间
- 为每个 VM 配置独立的内存区域
- 实现 VM 之间的内存隔离
- 控制敏感外设的访问

### 1.4.3 MPU 配置示例

```c
// MPU 区域配置结构
typedef struct {
    uint32_t base_addr;    // 区域基址
    uint32_t size;         // 区域大小
    uint32_t attributes;   // 属性：RW/RO/XN 等
    uint32_t access;       // 访问控制
} mpu_region_t;
```

## 1.5 集成中断控制器（GIC）

Cortex-R52 集成了 ARM Generic Interrupt Controller (GICv2)：

- **中断源**：支持最多 544 个中断源
- **优先级**：支持中断优先级嵌套
- **中断屏蔽**：支持中断使能/屏蔽控制
- **虚拟化支持**：支持虚拟中断注入

## 1.6 本章小结

本章介绍了 Cortex-R52 处理器的基本架构特性：

1. **ARMv8-R 架构**：64位架构支持，是 Cortex-R 系列首次
2. **多核支持**：分离/锁步模式，满足不同安全需求
3. **MPU**：增强的内存保护，支持虚拟机隔离
4. **GIC**：集成中断控制器，支持虚拟化

这些硬件特性为 Hypervisor 虚拟化提供了基础支持。下一章将详细介绍 ARMv8-R 的异常级别模型，这是理解 Hypervisor 工作原理的关键。

# 第1章 概述与基础概念

> 本章介绍Linux内存管理的基础概念，包括虚拟内存原理、物理内存管理、页表结构和MMU工作原理。

## 目录

## 1.1 虚拟内存概述

### 虚拟内存解决的问题

虚拟内存是操作系统提供的一种内存管理技术，它将虚拟地址空间映射到物理地址空间，使得应用程序可以使用比实际物理内存更大的地址空间。

**虚拟内存主要解决以下问题：**

1. **扩大地址空间**
   - 32位系统虚拟地址空间为4GB，64位系统更大
   - 应用程序无需关心物理内存是否足够

2. **内存保护**
   - 每个进程有独立的虚拟地址空间
   - 防止进程访问非法内存
   - 防止进程越界访问其他进程内存

3. **公平分配内存**
   - 操作系统统一管理物理内存
   - 按需分配给各个进程
   - 实现内存资源共享

4. **简化内存分配**
   - 应用程序看到的是连续的虚拟地址
   - 无需关心物理内存的碎片化
   - 简化了驱动程序和内核的开发

### 虚拟地址空间布局

Linux进程虚拟地址空间布局（32位）：

```
+------------------+ 0xFFFFFFFF
|     内核空间     | (1GB, 仅内核可访问)
+------------------+ 0xC0000000
|                  |
|    用户空间      |
|                  |
+------------------+ 0x08048000
|   未使用区域     |
+------------------+ 0x00000000
```

**各区域说明：**

| 区域 | 地址范围 | 说明 |
|------|----------|------|
| 内核空间 | 0xC0000000 - 0xFFFFFFFF | 内核态代码和数据 |
| 栈 | 向下增长 | 函数调用、局部变量 |
| 内存映射区 | - | mmap映射区域 |
| 堆 | 向上增长 | 动态内存分配(malloc) |
| BSS段 | - | 未初始化全局变量 |
| 数据段 | - | 已初始化全局变量 |
| 代码段 | - | 程序代码 |

Linux进程虚拟地址空间布局（64位）：

```
+------------------+ 0xFFFFFFFFFFFFFFFF
|     内核空间     |
+------------------+ 0xFFFF800000000000 (48位虚拟地址)
|                  |
|    用户空间      |
|                  |
+------------------+ 0x0000000000000000
```

**64位系统特点：**
- 实际只使用48位虚拟地址（256TB）
- 用户空间：0 - 0x00007FFFFFFFFFFF（128TB）
- 内核空间：0xFFFF800000000000 - 0xFFFFFFFFFFFFFFFF（128TB）

## 1.2 物理内存管理

### 内存模型

Linux支持三种内存模型：

1. **Flat Memory Model (平坦内存模型)**
   - 物理内存是连续的
   - 最简单的模型，适合大多数嵌入式系统
   - 节点(node)只有一个

2. **Discontiguous Memory Model (非连续内存模型)**
   - 物理内存不连续
   - 存在多个内存节点
   - 早期x86 NUMA系统使用

3. **Sparse Memory Model (稀疏内存模型)**
   - 物理内存可以是稀疏的
   - 支持内存热插拔
   - 当前主流模型

### 内存节点(NODE)

在NUMA(Non-Uniform Memory Access)系统中，不同CPU访问不同内存区域的延迟不同，因此划分为多个节点。

```c
// 节点结构定义 (include/linux/mmzone.h)
typedef struct pglist_data {
    struct zone node_zones[MAX_NR_ZONES];
    struct zonelist node_zonelists[MAX_ZONELISTS];
    int nr_zones;
    unsigned long node_start_pfn;
    unsigned long node_present_pages;
    unsigned long node_spanned_pages;
    // ...
} pg_data_t;
```

### 内存区域(ZONE)

每个节点划分为不同的内存区域：

| 区域 | 说明 | 典型大小 |
|------|------|----------|
| ZONE_DMA | DMA可访问的低地址内存 | 16MB |
| ZONE_DMA32 | 32位DMA可访问内存 | 512MB |
| ZONE_NORMAL | 正常可直接映射的内存 | 剩余 |
| ZONE_HIGHMEM | 高端内存(32位系统) | 可变 |
| ZONE_MOVABLE | 可移动内存(用于内存热插拔) | 可变 |

**32位系统内存布局示例：**

```
+------------------+ 4GB
|   ZONE_HIGHMEM   |
+------------------+ 896MB
|   ZONE_NORMAL    |
+------------------+ 16MB
|    ZONE_DMA      |
+------------------+ 0MB
```

## 1.3 页表基础

### 多级页表结构

Linux采用多级页表来管理虚拟地址到物理地址的映射：

**ARM64 5级页表：**

```
虚拟地址 (48位)
+----------+----------+----------+----------+----------+
|   PGD    |   PUD    |   PMD    |   PTE    |  Offset  |
|  16位    |  9位     |  9位      |  9位      |  12位    |
+----------+----------+----------+----------+----------+
    |          |          |          |          |
    v          v          v          v          v
  PGD表      PUD表      PMD表      PTE表     物理页面
(页全局目录) (页上级目录) (页中间目录) (页表项)  (4KB)
```

**x86_64 4级页表：**

```
虚拟地址 (48位)
+----------+----------+----------+----------+
|   PML4   |   PDPT   |    PD    |   PT     |
|  9位     |  9位     |  9位      |  9位      |
+----------+----------+----------+----------+
    |          |          |          |
    v          v          v          v
  PML4      PDPT       PD         PT
```

### 页表项格式

页表项(PTE)包含以下信息：

```
+----------+----------+----------+----------+----------+
| 物理页号  |  标志位  |  访问控制 |  其他标志  |
+----------+----------+----------+----------+

常见标志位：
- _PAGE_PRESENT: 页表项有效
- _PAGE_RW: 可写
- _PAGE_USER: 用户态可访问
- _PAGE_PWT: 写时直接更新(Write-Through)
- _PAGE_PCD: 禁用缓存(Cache Disable)
- _PAGE_ACCESSED: 已访问
- _PAGE_DIRTY: 已修改(Dirty)
```

### TLB工作原理

TLB(Translation Lookaside Buffer)是MMU中的高速缓存，用于缓存虚拟地址到物理地址的映射。

**TLB结构：**

```
TLB
+------------------------+
| 虚拟页号 | 物理页号 | 标志 |
+------------------------+
| 0x1000   | 0x3A000  | RWX  |
| 0x2000   | 0x5B000  | R X  |
| ...      | ...      | ...  |
+------------------------+
```

**TLB工作流程：**

1. CPU发出虚拟地址
2. MMU先查询TLB
3. 如果TLB命中(TLB hit)，直接获取物理地址
4. 如果TLB未命中(TLB miss)，查询页表
5. 将页表项加载到TLB
6. 返回物理地址

**TLB分类：**
- I-TLB: 指令TLB
- D-TLB: 数据TLB
- L2 TLB: 二级TLB

## 1.4 内存管理单元(MMU)

### MMU职责

内存管理单元(MMU)的主要职责：

1. **地址转换**
   - 将虚拟地址转换为物理地址
   - 通过页表完成映射

2. **访问控制**
   - 检查访问权限
   - 区分用户态和内核态访问

3. **内存保护**
   - 防止非法内存访问
   - 触发段错误/页错误异常

4. **缓存控制**
   - 控制页面的缓存策略
   - Write-Through / Write-Back

### MMU相关 ARM 指令

**启用MMU：**

```assembly
# 使能MMU (ARM64)
mov x0, #0
msr sctlr_el1, x0    # 先禁用MMU
dsb ish              # 数据同步屏障
isb                  # 指令同步屏障

# 设置TTBR0 (页表基址)
msr ttbr0_el1, x0

# 使能MMU
mov x0, #1
msr sctlr_el1, x0
dsb ish
isb
```

**TLB操作指令：**

```assembly
# 刷新TLB
tlbi vmalle1is       # 刷新所有TLB条目
dsb ish              # 数据同步
isb                  # 指令同步

# 刷新特定VA的TLB
tlbi vae1is, x0      # 刷新虚拟地址x0对应的TLB
```

## 本章小结

本章介绍了Linux内存管理的基础概念：

1. **虚拟内存**解决了地址空间扩展、内存保护、公平分配等问题
2. **物理内存管理**通过节点(NODE)和区域(ZONE)来组织
3. **多级页表**实现了高效的虚拟地址到物理地址的转换
4. **TLB**是MMU中的高速缓存，提高地址转换效率
5. **MMU**负责地址转换、访问控制和内存保护

这些概念是理解后续章节的基础，建议重点掌握。

# RCU - 面试题

## 基础概念

### Q1: RCU 的基本原理是什么？

**答：**
RCU（Read-Copy-Update）是读多写少场景下的高性能同步机制：

1. **读者**：使用 `rcu_read_lock/unlock` 保护临界区，无锁并发读取
2. **写者**：创建数据副本，修改副本，然后原子更新指针
3. **延迟释放**：旧数据在宽限期（Grace Period）结束后才释放

核心思想：读者无锁，写者通过复制和延迟删除避免冲突。

---

### Q2: 什么是宽限期（Grace Period）？

**答：**
宽限期是 RCU 等待所有 RCU 读者完成的等待时间。

```
时间线：
CPU0: ----R----R----          (RCU 读者)
CPU1: ----R---------R---    (RCU 读者)
CPU2:           --W--        (写者)
CPU3: --------------------   (空闲)

宽限期开始
        |
        |--- 所有 CPU 完成一次抢占 --|
                                |
                                |--- 宽限期结束，旧数据可释放
```

宽限期结束后，写者可以安全释放旧数据。

---

### Q3: RCU 与读写锁的区别？

**答：**

| 特性 | RCU | 读写锁 |
|------|-----|--------|
| 读者性能 | O(1)，无锁 | O(n)，需要加锁 |
| 写者性能 | 需要复制 | 需要复制 |
| 内存开销 | 较高（延迟释放） | 较低 |
| 写者阻塞读者 | 否 | 是 |
| 实现复杂度 | 较高 | 较低 |

---

### Q4: RCU 的优势与局限？

**答：**

**优势**：
1. 读者性能极佳，无锁读取
2. 写者不阻塞读者
3. 适合读多写少场景

**局限**：
1. 写者需要复制数据
2. 旧数据需要延迟释放
3. 不能用于写多读少场景
4. 实现复杂，调试困难

## 进阶问题

### Q5: synchronize_rcu 和 call_rcu 的区别？

**答：**

| 函数 | 行为 | 使用场景 |
|------|------|----------|
| `synchronize_rcu()` | 同步等待，阻塞直到宽限期结束 | 需要立即释放旧数据 |
| `call_rcu()` | 异步回调，宽限期结束后执行回调 | 不需要立即释放 |

```c
/* 同步等待 */
synchronize_rcu();
kfree(old_data);  /* 宽限期结束后才能执行 */

/* 异步回调 */
call_rcu(&old_data->rcu, free_data_callback);
/* 立即返回，宽限期结束后自动释放 */
```

---

### Q6: RCU 如何保证内存顺序？

**答：**
RCU 提供了隐式的内存顺序保证：

1. `rcu_assign_pointer`：提供写屏障
2. `rcu_dereference`：提供读屏障

```c
/* 写者 */
new = kmalloc(...);
new->value = 42;
rcu_assign_pointer(ptr, new);  /* 确保 value 写入在 ptr 更新之前 */

/* 读者 */
p = rcu_dereference(ptr);      /* 确保读取 ptr 在读取 *p 之前 */
if (p)
    print(p->value);            /* 读到的是新数据 */
```

---

### Q7: 什么场景适合使用 RCU？

**答：**
RCU 适合以下场景：

1. **读多写少**：90%+ 的读操作
2. **可容忍延迟读**：允许读到稍微过时的数据
3. **指针数据**：数据通过指针访问
4. **高性能需求**：需要低延迟读操作

典型应用：
- 路由表
- 配置文件
- 订阅者列表
- 监控统计

---

### Q8: SRCU 与 RCU 的区别？

**答：**

| 特性 | RCU | SRCU |
|------|-----|------|
| 临界区睡眠 | 禁止 | 允许 |
| 使用场景 | 进程/软中断上下文 | 任何上下文 |
| 性能开销 | 较低 | 较高 |
| 内存占用 | 较低 | 较高 |

## 实战问题

### Q9: RCU 使用的注意事项？

**答：**

1. **指针操作**：必须使用 `rcu_assign_pointer` 和 `rcu_dereference`
2. **内存释放**：旧数据必须等待宽限期结束
3. **临界区**：确保 `rcu_read_lock/unlock` 配对
4. **链表操作**：使用 `list_add_rcu`、`list_del_rcu` 等

---

### Q10: 如何调试 RCU 问题？

**答：**

1. **RCU 调试选项**：
   ```
   CONFIG_RCU_TRACE
   CONFIG_RCU_CPU_STALL_INFO
   ```

2. **查看 RCU 状态**：
   ```bash
   cat /sys/kernel/debug/rcu/rcu0/rcudata
   cat /sys/kernel/debug/rcu/rcu0/rcugp
   ```

3. **检测 stall**：
   ```bash
   # 如果 RCU 读者阻塞太久，内核会输出警告
   dmesg | grep -i "RCU"
   ```

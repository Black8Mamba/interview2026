# RCU (Read-Copy-Update)

## 概述

RCU（Read-Copy-Update）是 Linux 内核中一种高性能的同步机制，专为"读多写少"的场景设计。RCU 的核心思想是：读者可以无锁地并发读取共享数据，写者在修改数据时创建数据副本，延迟释放旧数据。

RCU 的主要特点：
- **读者无锁**：读取操作无需加锁，无性能损失
- **写者延迟删除**：旧数据在宽限期后才释放
- **高性能**：在读多写少场景下性能优异

## RCU 的基本原理

### 1. 读取阶段
- 读者使用 `rcu_read_lock()` 和 `rcu_read_unlock()` 保护临界区
- 读者可以无限制地并发读取
- 读者不会阻塞写者

### 2. 更新阶段
- 写者创建数据的副本
- 修改副本
- 使用原子操作更新指针
- 调用 `synchronize_rcu()` 或 `call_rcu()` 等待宽限期结束
- 释放旧数据

### 3. 宽限期（Grace Period）
- 宽限期是所有 RCU 读者完成的等待时间
- 宽限期结束后，旧数据可以被安全释放

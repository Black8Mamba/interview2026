# 内存屏障

## 概述

内存屏障（Memory Barrier）是 Linux 内核中用于控制 CPU 和编译器的内存访问顺序的机制。在现代处理器中，为了提高性能，CPU 和编译器会对内存访问进行各种优化，包括：

1. **编译器优化**：重新排列代码中的内存访问顺序
2. **CPU 乱序执行**：CPU 指令流水线可能不按程序顺序执行
3. ** CPU 缓存**：多核处理器的缓存一致性协议可能导致内存访问顺序变化

内存屏障用于确保在并发环境下，对共享数据的访问顺序符合程序员的预期。

## 为什么需要内存屏障

考虑以下场景：

```c
/* 线程 A */
data = 42;           /* 1. 写入数据 */
flag = 1;            /* 2. 写入标志 */

/* 线程 B */
if (flag == 1) {     /* 3. 读取标志 */
    print(data);     /* 4. 读取数据 */
}
```

在缺乏内存屏障的情况下，编译器或 CPU 可能重排操作顺序，导致线程 B 可能读到 `data` 的旧值。

## 内存屏障的类型

1. **编译器屏障**：阻止编译器重排代码
2. **硬件内存屏障**：确保 CPU 执行顺序
3. **内存顺序原语**：提供特定顺序保证

## 典型使用场景

1. **自旋锁**：获取锁后的内存访问
2. **无锁算法**：CAS 操作前后的顺序保证
3. **设备驱动**：MMIO 寄存器访问顺序
4. **中断处理**：中断上下文与进程上下文的数据同步

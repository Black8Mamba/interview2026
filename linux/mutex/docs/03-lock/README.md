# 锁机制

## 概述

锁是 Linux 内核中最常用的同步机制，用于保护临界区（Critical Section），确保在任意时刻只有一个执行上下文可以访问共享资源。

Linux 内核提供了多种锁机制，每种锁适用于不同的场景：
- **自旋锁**：适用于短临界区、不可睡眠的场景
- **互斥锁**：适用于需要睡眠的临界区
- **信号量**：支持计数的锁，可用于资源计数
- **读写锁**：读多写少场景的优化
- **顺序锁**：读多写少且可以容忍偶尔读到脏数据

## 自旋锁 (Spinlock)

自旋锁的核心思想是"忙等待"——当锁不可用时，CPU 循环检查锁是否可用。自旋锁适用于：
- 临界区非常短
- 持有锁期间不会睡眠（不可中断上下文）

**注意**：自旋锁不能递归持有，否则会导致死锁。

## 互斥锁 (Mutex)

互斥锁是一种睡眠锁。当锁不可用时，进程会进入睡眠状态，让出 CPU。自旋锁适用于：
- 临界区较长
- 可以在锁等待时睡眠（进程上下文）

## 信号量 (Semaphore)

信号量是一种计数锁，可以支持多个并发访问。适用于：
- 资源有多个实例
- 需要计数控制

## 读写锁 (Rwlock)

读写锁允许多个读者同时访问，但写者是互斥的。适用于：
- 读多写少的场景

## 顺序锁 (Seqlock)

顺序锁是一种写者优先的锁，适用于：
- 读多写少
- 写操作非常快
- 可以容忍偶尔读到不一致的数据

## 选择锁的指导原则

| 场景 | 推荐锁 |
|------|--------|
| 中断上下文、短临界区 | 自旋锁 |
| 进程上下文、长临界区 | 互斥锁 |
| 多个资源实例 | 信号量 |
| 读多写少 | 读写锁 / RCU |
| 写操作极快的读多写少 | 顺序锁 |

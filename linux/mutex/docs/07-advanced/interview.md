# 高级主题 - 面试题

## lockdep

### Q1: lockdep 的作用是什么？

**答：**
lockdep 是 Linux 内核的运行时锁正确性检查器，主要功能：

1. **检测潜在死锁**：分析锁获取顺序，检测可能导致死锁的环路
2. **递归锁检测**：检测同一把锁的递归获取
3. **锁睡眠检测**：检测在持锁状态下睡眠
4. **释放错误检测**：检测未持有锁时的解锁操作

---

### Q2: lockdep 的工作原理？

**答：**

1. **记录锁获取序列**：每次获取锁时，记录锁和调用点
2. **构建依赖图**：根据历史构建锁依赖图
3. **环路检测**：检测图中是否存在环路
4. **报告问题**：发现潜在死锁时输出警告

---

### Q3: 如何使用 lockdep 调试锁问题？

**答：**

**启用 lockdep**：
```bash
# 内核启动参数
lockdep=1
```

**触发检测**：运行可能触发死锁的代码

**查看输出**：
```bash
cat /proc/lockdep
cat /proc/lock_stat
```

## 综合问题

### Q4: Linux 内核中有哪些同步机制？如何选择？

**答：**

| 机制 | 适用场景 |
|------|----------|
| 原子操作 | 简单计数器、标志位 |
| 自旋锁 | 短临界区、中断上下文 |
| 互斥锁 | 长临界区、进程上下文 |
| 信号量 | 资源池、生产者-消费者 |
| 读写锁 | 读多写少 |
| RCU | 读极多写极少 |
| Per-CPU | CPU 本地数据 |
| Futex | 用户态高性能锁 |

**选择原则**：
- 读多写少 → RCU / 读写锁
- 短临界区 → 自旋锁
- 长临界区 → 互斥锁
- 资源计数 → 信号量 / 原子操作

---

### Q5: 如何调试内核中的死锁问题？

**答：**

1. **使用 lockdep**：静态分析锁顺序
2. **SysRq**：打印锁信息
   ```bash
   echo t > /proc/sysrq-trigger
   ```
3. **perf lock**：分析锁性能
   ```bash
   perf lock record <cmd>
   perf lock report
   ```
4. **ftrace**：跟踪锁事件
   ```bash
   echo 'lock:*' > /tracing/set_event
   ```
5. **crash**：分析内核转储

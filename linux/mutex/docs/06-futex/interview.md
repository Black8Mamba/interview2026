# Futex - 面试题

## 基础概念

### Q1: Futex 的工作原理？

**答：**
Futex（Fast Userspace Mutex）结合用户态自旋和内核等待：

1. **快速路径**：尝试用 CAS 在用户态获取锁，成功则返回
2. **慢速路径**：锁被占用时，在用户态自旋一段时间
3. **阻塞路径**：自旋失败后，调用 futex() 进入内核等待
4. **唤醒**：锁持有者释放时，通过 futex() 唤醒等待者

---

### Q2: 为什么 Futex 比传统系统调用快？

**答：**

1. **避免不必要的系统调用**：无竞争时完全在用户态完成
2. **减少内核参与**：只在真正需要等待时进入内核
3. **高效的唤醒机制**：内核直接操作等待队列

---

### Q3: Futex 与普通互斥锁的区别？

**答：**

| 特性 | Futex | 普通系统调用 mutex |
|------|-------|---------------------|
| 无竞争 | 用户态 CAS | 系统调用 |
| 竞争 | futex 系统调用 | 系统调用 |
| 性能 | 更高 | 标准 |
| 实现 | 内核支持 | 用户态库 |

---

## 进阶问题

### Q4: Futex 的状态含义？

**答：**

```
- 0: 锁未被持有
- 1: 锁被持有（无等待者）
- 2+: 锁被持有，且有等待者
```

---

### Q5: Futex 的应用场景？

**答：**

Futex 是以下用户态同步机制的底层实现：

1. **pthread_mutex**：POSIX 互斥锁
2. **pthread_condvar**：条件变量
3. **semaphore**：POSIX 信号量
4. **其他用户态锁**

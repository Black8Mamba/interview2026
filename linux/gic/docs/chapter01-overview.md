# 第1章 中断子系统概述

本章将介绍Linux内核中断子系统的基本概念、模型演进和分类方式，为后续深入学习中断处理流程、ARM64架构特性和GIC中断控制器奠定理论基础。

## 1.1 中断基本概念

### 1.1.1 中断的定义

**中断（Interrupt）** 是计算机系统中一种重要的异步事件处理机制。当外部硬件设备需要处理器注意时，会通过中断信号通知CPU，暂停当前正在执行的指令流，切换到特定的中断处理程序（Interrupt Handler）去处理紧急事务，处理完毕后再恢复执行被中断的代码。

在ARM64架构中，中断通常通过GIC（Generic Interrupt Controller）中断控制器进行管理。当外设发出中断请求（Interrupt Request，IRQ）时，GIC将其转发给目标CPU核心，触发异常级别（Exception Level）的切换，进入EL1（内核态）或EL0（用户态）的中断处理入口。

### 1.1.2 中断的作用

中断机制的主要作用体现在以下几个方面：

1. **提高CPU利用率**：无需让CPU轮询等待外设状态，设备准备好数据后主动通知CPU，避免了CPU资源的空转消耗。

2. **实现实时响应**：对于时间敏感的外设（如网络包到达、磁盘数据传输完成），中断机制确保能够及时响应，避免数据丢失。

3. **降低系统功耗**：在无中断时CPU可以进入低功耗状态，只有在需要处理时才被唤醒，这对于移动设备的续航至关重要。

### 1.1.3 中断与轮询的对比

| 特性 | 中断方式 | 轮询方式 |
|------|----------|----------|
| CPU占用 | 空闲时低负载，事件驱动 | 持续占用CPU循环检查 |
| 响应延迟 | 低延迟，事件触发即处理 | 取决于轮询周期 |
| 功耗 | 低，CPU可休眠 | 高，CPU持续运行 |
| 实现复杂度 | 较高，需要中断处理程序 | 简单，顺序读取状态 |
| 适用场景 | 事件稀疏、实时性要求高 | 事件频繁、延迟容忍度高 |

在实际系统中，中断和轮询往往结合使用。例如，NIC（网络接口卡）驱动程序中，中断用于通知新数据包到达，而读取数据的过程可能采用轮询或NAPI（New API）混合模式来提高批量处理效率。

## 1.2 Linux中断模型

### 1.2.1 顶半部与底半部模型

Linux内核的中断处理采用**顶半部（Top Half）/底半部（Bottom Half）** 分离的设计模式，目的是将中断处理分为两个阶段：

- **顶半部（中断处理程序）**：在中断禁用状态下执行，负责最紧急、最短时间的处理工作，如读取硬件状态、清除中断标志等。顶半部必须快速完成，因为此时CPU处于中断禁止状态，其他中断可能被延迟。

- **底半部**：在中断使能后执行，负责耗时较长的数据处理工作，如网络数据包的协议栈处理、磁盘I/O的数据拷贝等。

这种设计模式的演进过程如下：

```
早期版本 → 2.4内核 → 2.6内核 → 现代版本
  BH        Softirqs   Tasklets   Workqueues/线程化中断
```

### 1.2.2 底半部机制的演进

1. **BH（Bottom Half）**：早期Linux使用的底半部机制，通过`mark_bh()`注册回调，在`do_BOTTOM_HALF()`中执行。由于并发能力有限，已被淘汰。

2. **Softirq**：2.6内核引入的软中断机制，运行在中断上下文中，支持多CPU并发执行。常见类型包括`NET_TX_SOFTIRQ`、`NET_RX_SOFTIRQ`、`TIMER_SOFTIRQ`等。

3. **Tasklet**：基于softirq封装的机制，同一类型的tasklet只能在单个CPU上串行执行，但不同类型的tasklet可以并发。实现简单，是驱动开发中最常用的底半部方式。

4. **Workqueue**：将工作延迟到进程上下文执行，可以睡眠，适合需要阻塞操作的处理逻辑。

5. **线程化中断（Threaded IRQ）**：Linux 3.0引入的特性，允许将中断处理完全放在内核线程中运行，支持优先级设置和CPU亲和性控制，提高了实时性。

### 1.2.3 现代中断处理模型

在Linux 6.x版本中，典型的中断处理流程如下：

```c
// 伪代码展示中断处理流程
static irqreturn_t my_device_irq(int irq, void *dev_id)
{
    // 顶半部：快速完成
    if (!my_device_has_data())
        return IRQ_NONE;  // 伪中断

    // 清除中断状态，读取关键数据
    my_device_clear_interrupt();
    my_device_read_data();

    // 调度底半部
    if (need_schedule_bottom_half())
        tasklet_schedule(&my_tasklet);

    return IRQ_HANDLED;
}

static void my_device_tasklet(struct tasklet_struct *t)
{
    // 底半部：耗时处理
    process_data();
}
```

## 1.3 中断分类

### 1.3.1 硬件中断

硬件中断由外设主动发起，通过中断控制器（如GIC）传递给CPU。在ARM64系统中：

- **IRQ（Interrupt Request）**：普通中断，用于大多数外设，如UART、Timer、GPIO等。
- **FIQ（Fast Interrupt Request）**：快速中断，在ARM架构中具有更高优先级，可抢占IRQ处理。现代Linux通常将FIQ用于安全监控或其他特殊用途。

GIC中断控制器支持的中断类型包括：
- **SGI（Software Generated Interrupt）**：软件生成中断，用于多核间通信。
- **PPI（Peripheral Private Interrupt）**：私有外设中断，如每个CPU的本地定时器。
- **SPI（Shared Peripheral Interrupt）**：共享外设中断，可路由到任意CPU核。

### 1.3.2 软件中断

软件中断不是由硬件触发，而是由软件主动触发：

- **软中断（Softirq）**：内核用于处理延迟工作的机制，如网络协议栈的包处理。
- **系统调用**：在ARM64上通过`SVC`（Supervisor Call）指令触发，从用户态切换到内核态执行系统服务。

### 1.3.3 异常

异常是CPU在执行指令时检测到的错误条件，需要内核立即处理：

- **同步异常**：由当前指令执行触发，如未定义指令（UND）、数据中止（DABT）、指令中止（IABT）、特权错误（SVC/HVC/SMC）。
- **异步异常**：不由指令触发，如IRQ、FIQ、系统错误（SError）。

在ARM64架构中，同步异常通过`ESR_ELx`（Exception Syndrome Register）寄存器记录异常原因和触发指令信息。

### 1.3.4 伪中断

伪中断（Spurious Interrupt）是指中断控制器报告的中断但实际没有对应的外设中断请求发生。可能原因包括：

1. 外设在中断处理完成前已清除中断源
2. 电气干扰或信号完整性问题
3. 中断控制器配置错误

Linux内核的中断处理程序会检查中断是否真实有效，对于无法识别的中断返回`IRQ_NONE`，避免伪中断影响系统稳定性。

## 本章面试题

### 1. 什么是中断？中断和轮询有什么区别？

**参考答案**：中断是外设通知CPU的异步事件机制，CPU暂停当前任务转去处理中断。区别在于：中断是事件驱动，CPU无需空转等待，功耗低、响应及时；轮询需要CPU持续检查设备状态，实现简单但资源浪费大、实时性差。

### 2. 为什么要将中断处理分为顶半部和底半部？

**参考答案**：因为中断处理程序在中断禁用状态下执行，时间过长会阻塞其他中断。顶半部处理紧急但简短的操作（如清除中断标志），底半部处理耗时操作（如数据处理），两者分离保证系统响应能力和处理完整性的平衡。

### 3. Linux中的底半部机制有哪些？它们有什么区别？

**参考答案**：主要有Softirq、Tasklet、Workqueue和线程化中断。Softirq在中断上下文运行，支持多核并发；Tasklet基于Softirq，同类型串行执行；Workqueue在进程上下文运行，可以睡眠；线程化中断将整个处理放到内核线程，支持优先级和CPU亲和性。

### 4. ARM64中的异常级别（EL）是如何与中断处理关联的？

**参考答案**：Linux运行在EL1（内核态），用户程序运行在EL0。当外设触发IRQ时，CPU自动切换到EL1，保存现场后执行中断向量表中的处理代码。EL2用于虚拟化，EL3用于安全世界监控。

### 5. 什么是伪中断？如何处理？

**参考答案**：伪中断是中断控制器报告但实际无有效中断请求的情况。可能由外设状态变化、电气干扰或配置错误导致。处理方法是中断处理程序检查中断有效性，返回`IRQ_NONE`让内核忽略该中断。

### 6. GIC中断控制器支持哪些中断类型？

**参考答案**：GIC支持三类中断：SGI（软件生成中断，用于多核通信）、PPI（私有外设中断，如本地定时器）、SPI（共享外设中断，可路由到任意CPU）。此外还分为边沿触发和电平触发两种类型。

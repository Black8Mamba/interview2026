## 2. 时序计算公式

### 2.1 SPI时钟相关计算

#### 最大数据传输率

```
SDR模式:
    Throughput = fCLK × 1 / 8 × N × D

    fCLK: SPI时钟频率
    N: 线数 (1=SPI, 2=DSPI, 4=QSPI, 8=Octal)
    D: 每字节所需周期数 (通常为1)

示例: W25Q128JV @ 133MHz QSPI
    Throughput = 133MHz × 4 / 8 = 66.5 MB/s
```

#### DTR模式有效频率

```
DTR (Double Transfer Rate) 模式:
    有效频率 = fCLK × 2

示例: MT35XU512ABA @ 200MHz DTR
    有效频率 = 400MHz
    带宽 = 400MB/s (4字节对齐)
```

### 2.2 读取延迟计算

#### 命令到数据延迟

```
tRD = (DC + CO) / fCLK

    DC: Dummy Cycle数量
    CO: Command Opcode周期 (通常为1)
    fCLK: 时钟频率

示例: W25Q128JV @ 133MHz, DC=8
    tRD = (8 + 1) / 133MHz = 67.7ns
```

#### 实际读取时间计算

```
读取时间 = tCMD + tADDR + tDUMMY + tDATA

    tCMD: 命令周期时间 = 1 / fCLK
    tADDR: 地址周期时间 = 24bit / (N × fCLK)
    tDUMMY: 等待周期时间 = DC / fCLK
    tDATA: 数据周期时间 = Bytes / (N × fCLK)

示例: 读取256字节, W25Q128JV @ 133MHz QSPI
    tCMD = 1/133MHz = 7.5ns
    tADDR = 24bit / (4 × 133MHz) = 45ns
    tDUMMY = 8/133MHz = 60ns
    tDATA = 256Byte / (4 × 133MHz) = 481ns
    总时间 = 7.5 + 45 + 60 + 481 = 594ns
```

### 2.3 编程时间计算

#### 页编程时间

```
tPP = tPP_base + (Bytes / 256) × tPP_per_page

    tPP_base: 基础编程时间
    tPP_per_page: 每页额外时间

示例: W25Q128JV编程256字节
    tPP = 0.5ms + 1 × 0.5ms = 1ms (典型值)
```

#### 连续编程优化

```
最佳连续编程数 = Sector_Size / Page_Size

    SPI Flash通常: 4KB sector, 256B page
    最佳连续编程: 4096 / 256 = 16页

注意: 跨页编程需要重新发送命令
```

### 2.4 擦除时间计算

#### 擦除时间估算

```
tERASE = N × tUNIT × Factor

    N: 擦除单元数量
    tUNIT: 单元基础时间
    Factor: 温度/电压影响因子

温度影响因子:
    -40°C ~ +25°C: 1.0
    +25°C ~ +85°C: 0.8
    +85°C ~ +125°C: 0.6

示例: W25Q128JV擦除256个4KB扇区
    tERASE = 256 × 80ms × 0.8 = 16.4s (典型)
    tERASE_max = 256 × 200ms × 1.0 = 51.2s (最大)
```

### 2.5 时序余量计算

#### 建立/保持时间余量

```
Setup Margin = t - tSU
Hold Margin = t - tH

    t: 实际时钟周期
    tSU: 设备要求的最小建立时间
    tH: 设备要求的最小保持时间

安全阈值: Margin > 20% 周期时间

示例: 100MHz时钟 (10ns周期)
    tDS (要求) = 2ns
    tDH (要求) = 2ns
    Setup Margin = 10ns - 2ns = 8ns (80%)
    Hold Margin = 10ns - 2ns = 8ns (80%)
    安全!
```

### 2.6 总线带宽计算

#### 多通道总线利用

```
实际带宽 = 标称带宽 × 利用率 × 协议开销

    利用率: 实际数据传输时间/总时间
    协议开销: 命令+地址+状态等非数据时间

典型利用率:
    顺序读取: 95%
    随机读取: 70%
    连续写入: 90%

示例: 133MHz QSPI顺序读取
    标称带宽: 133MHz × 4 / 8 = 66.5 MB/s
    实际带宽: 66.5 × 0.95 × 0.9 = 56.9 MB/s
```

## 3. 时序配置指南

### 3.1 SPI时钟配置

| 时钟范围 | 适用场景 | 配置建议 |
|----------|----------|----------|
| 0-20MHz | 低速模式, 兼容性优先 | 调试/老设备 |
| 20-50MHz | 标准模式, 普通应用 | 推荐配置 |
| 50-80MHz | 高速模式, 性能优先 | XIP启动 |
| 80-133MHz | 极速模式, 高端MCU | 大容量读取 |
| 133-200MHz | Octal/DTR模式 | 高性能系统 |

### 3.2 典型时序配置示例

#### STM32 QSPI配置示例

```c
// W25Q128JV @ 133MHz QSPI
QSPI_CommandTypeDef Cmd;

Cmd.InstructionMode   = QSPI_INSTRUCTION_4_LINES;
Cmd.Instruction      = 0xEB;  // QUAD IO READ
Cmd.AddressMode      = QSPI_ADDRESS_4_LINES;
Cmd.AddressSize      = QSPI_ADDRESS_24_BITS;
Cmd.AlternateByteMode = QSPI_ALTERNATE_BYTES_4_LINES;
Cmd.AlternateBytes   = 0x00; // 4字节mode
Cmd.AlternateBytesSize = QSPI_ALTERNATE_BYTES_8_BITS;
Cmd.DataMode         = QSPI_DATA_4_LINES;
Cmd.NbOfData         = 256;
Cmd.DummyCycles      = 6;
Cmd.SIOOMode         = QSPI_SIOO_INST_EVERY_CMD;
```

#### FPGA并行接口时序

```verilog
// 异步读取时序 (90ns)
parameter tACC = 90;
parameter tCO  = 90;
parameter tOH  = 0;

// 读取时序控制
always @(posedge clk) begin
    if (cs_n == 0 && we_n == 1) begin
        addr_latch <= addr;
        #tACC data_out <= flash_data;
    end
end
```

### 3.3 时序问题排查

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 读取数据错误 | 时钟超速 | 降低时钟频率 |
| 读取不稳定 | 数据建立/保持不足 | 调整采样边沿 |
| 写入失败 | 写使能未生效 | 确认0x06命令执行 |
| 擦除失败 | 等待时间不足 | 增加轮询超时 |
| 随机错误 | 信号完整性 | 检查PCB布局 |

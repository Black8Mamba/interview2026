# Nor Flash 存储结构与寻址

理解 Nor Flash 的内部存储结构和地址映射机制，是正确使用和开发 Nor Flash 驱动的基础。本章将详细介绍 Nor Flash 的芯片内部组织、扇区结构类型、地址映射原理以及特殊存储区域。

---

## 芯片内部组织结构

Nor Flash 的内部架构从底层存储单元到顶层地址映射，形成了层次化的组织结构。理解这一结构有助于开发者更好地设计存储管理方案。

### 存储阵列（Array）

Nor Flash 的核心是存储阵列（Array），由大量浮栅晶体管（Floating Gate Transistor）排列成矩阵结构。每个存储单元存储 1-bit 数据，通过浮栅中捕获的电荷量来区分 "0" 和 "1" 状态。

**存储单元的工作原理**：

- **擦除后**：浮栅中无电荷，读取时位线检测到电流通过，表示数据 "1"（初始状态）
- **编程后**：浮栅中注入电荷，产生的电场削弱或阻断位线电流，表示数据 "0"

存储阵列的基本组织方式如下：

```
                    ┌─────────────────────────────────────┐
                    │           Storage Array             │
                    │  ┌─────────────────────────────┐    │
                    │  │  Sector 0 (4KB/32KB/64KB)  │    │
                    │  ├─────────────────────────────┤    │
                    │  │  Sector 1                   │    │
                    │  ├─────────────────────────────┤    │
                    │  │  ...                        │    │
                    │  ├─────────────────────────────┤    │
                    │  │  Sector N                   │    │
                    │  └─────────────────────────────┘    │
                    └─────────────────────────────────────┘
```

**存储阵列的关键参数**：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| 存储单元数 | 芯片总存储单元数量 | 取决于容量 |
| 单元排列 | 行（Word Line）× 列（Bit Line） | 矩阵形式 |
| 访问方式 | 随机字节访问 | 支持 |
| 数据保持 | 断电后数据保持时间 | 10-20 年 |

### 扇区（Sector）与块（Block）概念

扇区（Sector）和块（Block）是 Nor Flash 中用于组织存储空间的基本单位。虽然这两个术语有时可以互换使用，但在不同厂商和不同类型 Flash 中有不同的含义。

**基本概念**：

- **扇区（Sector）**：Flash 芯片的最小擦除单位。不同芯片的扇区大小可能不同，常见的有 4KB、32KB、64KB 等。
- **块（Block）**：通常与扇区概念相同，指的是擦除操作的最小单位。但在某些文档中，"块" 可能指多个扇区的组合。

**为什么需要分扇区**：

Flash 存储器的擦除操作有其物理特性：
- 擦除操作以块为单位进行，无法擦除单个字节或字
- 擦除操作会重置整个块内的所有位为 "1"
- 擦除操作耗时较长（毫秒级），需要合理设计擦除策略

**擦除操作的特性**：

```
擦除前（块内数据）：11111111 11111111 11111111 11111111
执行块擦除后：      00000000 00000000 00000000 00000000
写入数据后：        01010110 11001010 00111001 10101010
```

### 页（Page）概念

页（Page）是 SPI Nor Flash 编程操作的最小单位。与 Nand Flash 不同，Nor Flash 通常没有严格的页概念，但在实际编程操作中仍存在一些约束。

**SPI Nor Flash 的页编程**：

- **页大小**：通常为 256 字节（部分芯片支持 512 字节）
- **页编程限制**：单次编程操作不能超过一页范围
- **跨页处理**：需要分多次编程或使用连续编程命令

**页编程示意图**：

```
地址范围        0x00    0x01    0x02    ...   0xFF（一页）
                ┌──────┬──────┬──────┬─────┬──────┐
单页编程范围    │ Byte │ Byte │ Byte │ ... │ Byte │
                │  0   │  1   │  2   │     │ 255  │
                └──────┴──────┴──────┴─────┴──────┘

跨页写入需要：
Page 0: 0x00 - 0xFF
Page 1: 0x100 - 0x1FF
...依次类推
```

**注意**：Nor Flash 的页编程限制与 Nand Flash 不同。Nor Flash 可以随机地址写入（无需先擦除），但为了保证写入可靠性和性能，通常仍建议以页为单位进行编程。

### 字节/字组织方式

Nor Flash 中的数据可以按字节（Byte）或字（Word）为单位进行组织和访问。

**数据组织方式**：

| 组织方式 | 数据宽度 | 说明 |
|----------|----------|------|
| x8 模式 | 8-bit | 单字节访问，适用于成本敏感应用 |
| x16 模式 | 16-bit | 双字节访问，提高数据传输效率 |
| x32 模式 | 32-bit | 四字节访问，并行 Nor Flash 常见 |

**并行 Nor Flash 的位宽选择**：

```
x8 模式连接：                      x16 模式连接：
┌─────────────┐                   ┌─────────────┐
│   Nor Flash │                   │   Nor Flash │
│  D[7:0]     │◄─── 8-bit ───────►│  D[15:0]    │
│  A[x:0]     │◄─── Address ─────►│  A[x:1]     │
└─────────────┘                   └─────────────┘
```

**访问单位对性能的影响**：

- **字节访问**：灵活但效率较低
- **字访问**：效率更高，适合大数据量传输
- **突发访问**：连续地址的批量读取，提高传输效率

## 扇区/块结构类型

Nor Flash 芯片根据应用场景的不同，采用不同的扇区结构设计。理解这些结构有助于选择合适的芯片和设计存储管理方案。

### 均匀扇区（Uniform Sector）

均匀扇区（Uniform Sector）是指芯片内所有扇区大小相同的结构。这是 SPI Nor Flash 最常见的扇区组织方式。

**特点**：

- 所有扇区大小完全一致
- 便于统一管理，简化软件设计
- 擦除时间一致，性能可预测
- 适合需要灵活分区的应用

**典型均匀扇区结构**：

| 容量 | 扇区大小 | 扇区数量 | 典型芯片 |
|------|----------|----------|----------|
| 8Mbit | 64KB | 16 | W25Q80DV |
| 16Mbit | 64KB | 32 | W25Q128 |
| 32Mbit | 64KB | 64 | W25Q256 |
| 64Mbit | 64KB | 128 | W25Q512 |

**均匀扇区布局图**：

```

│ 0x010000 - 0x01FFFF │ Sector 1 (64KB)                  │
├─────────────────────┤

│                     │                                   │
├─────────────────────┤
│ 0x7F0000 - 0x7FFFFF │ Sector 127 (64KB)                │
└────────────────────────────────────────────────────────┘
```

### 非均匀扇区（Non-uniform Sector）

非均匀扇区（Non-uniform Sector）是指芯片内包含不同大小扇区的结构。这种设计通常是为了满足特定应用需求，如启动代码存储和文件系统分区。

**特点**：

- 包含多种规格的扇区
- 通常小扇区用于启动代码，大扇区用于数据存储
- 软件设计更复杂，需要针对不同扇区别管理
- 常见于需要支持多种启动模式的芯片

## 地址映射机制

Nor Flash 的地址映射机制决定了软件如何访问芯片内的存储单元。理解地址编码方式、寻址模式和对齐要求，对于正确编写驱动和应用程序至关重要。

### 地址编码方式

Nor Flash 使用线性地址空间，从地址 0x000000 开始递增。不同容量的芯片使用不同宽度的地址总线。

**SPI Nor Flash 地址编码**：

SPI Nor Flash 通常使用 3 字节（24 位）或 4 字节（32 位）地址：

| 地址模式 | 地址位数 | 最大容量 | 命令格式 |
|----------|----------|----------|----------|
| 3-byte | 24-bit | 16MB (128Mbit) | A[23:0] |
| 4-byte | 32-bit | 最高 4GB | A[31:0] |

**3 字节地址模式**：

```
W25Q128 (16MB) 地址编码：
┌────────────────────────────┐
│  A[23:16] │ A[15:8] │ A[7:0] │
│   高字节   │  中字节  │  低字节 │
└────────────────────────────┘
地址范围：0x000000 - 0xFFFFFF (16MB)
```

**4 字节地址模式**：

```
W25Q256 (32MB) 地址编码：
┌────────────────────────────────────┐
│ A[31:24] │ A[23:16] │ A[15:8] │ A[7:0] │
│ 扩展字节  │  高字节   │  中字节  │  低字节  │
└────────────────────────────────────┘
地址范围：0x00000000 - 0x01FFFFFF (32MB)
```

**地址模式切换**：

许多大容量 SPI Nor Flash 支持在 3 字节和 4 字节地址模式之间切换：

```c
// 进入 4 字节地址模式
void enter_4byte_mode(uint8_t flash_id) {
    if (flash_id == WINBOND_W25Q256) {
        write_enable();
        send_command(0xB7);  // Enter 4-Byte Address Mode
    }
}

// 退出 4 字节地址模式
void exit_4byte_mode(uint8_t flash_id) {
    if (flash_id == WINBOND_W25Q256) {
        write_enable();
        send_command(0xE9);  // Exit 4-Byte Address Mode
    }
}
```

### 字节寻址 vs 字寻址

Nor Flash 支持字节级和字级（Word，16-bit 或 32-bit）寻址方式。

**字节寻址（Byte Addressing）**：

- 最基本的寻址方式，每个地址对应一个字节
- 灵活性高，可访问任意单字节数据
- SPI Nor Flash 主要使用字节寻址

**字寻址（Word Addressing）**：

- 每个地址对应 2 字节（16-bit）或 4 字节（32-bit）
- 主要用于并行 Nor Flash（x16 或 x32 位宽）
- 可以提高数据传输效率

**不同位宽下的地址计算**：

```
x8 模式（字节寻址）：
地址 0x0000 → 字节 0
地址 0x0001 → 字节 1
...

x16 模式（字寻址）：
地址 0x0000 → 字节 0-1（低字节在低地址）
地址 0x0001 → 字节 2-3
...

x32 模式（双字寻址）：
地址 0x0000 → 字节 0-3
地址 0x0001 → 字节 4-7
...
```

### 地址对齐要求

Nor Flash 在某些操作中对地址对齐有特定要求。

**读取操作的地址对齐**：

- 字节读取：无对齐要求
- 字读取（x16 模式）：偶数地址对齐
- 突发读取：建议 4 字节或 8 字节对齐以获得最佳性能

**编程操作的地址对齐**：

- 字节编程：无对齐要求
- 页编程：必须从页起始地址开始
- 跨页编程：需要分多次进行

**擦除操作的地址对齐**：

- 扇区擦除：必须对齐到扇区边界
- 块擦除：必须对齐到块边界
- 整片擦除：无对齐要求

**地址对齐示例**：

```
64KB 扇区对齐要求（起始地址 0x10000）：
有效擦除地址：0x10000, 0x20000, 0x30000, ...
无效擦除地址：0x15000, 0x1FFFF, 0x25000, ...

页对齐要求（256 字节页，起始地址 0x100）：
有效编程起始地址：0x100, 0x200, 0x300, ...
无效编程起始地址：0x101, 0x1FF, 0x250, ...
```

### 大端模式 vs 小端模式

数据的字节序（Endianness）影响多字节数据在存储器和 CPU 之间的解释方式。

**大端模式（Big Endian）**：

- 高字节存储在低地址
- 符合人类阅读习惯
- 网络协议常用

```
大端存储（0x12345678）：
地址 0x00: 0x12
地址 0x01: 0x34
地址 0x02: 0x56
地址 0x03: 0x78
```

**小端模式（Little Endian）**：

- 低字节存储在低地址
- CPU 内部常用（x86、ARM）
- 更高效的数据访问

```
小端存储（0x12345678）：
地址 0x00: 0x78
地址 0x01: 0x56
地址 0x02: 0x34
地址 0x03: 0x12
```

**Nor Flash 的字节序**：

- 大多数 Nor Flash 采用小端模式存储
- 部分并行 Nor Flash 可通过模式引脚配置
- SPI Nor Flash 通常以字节为单位传输，不涉及字顺序问题

**跨平台数据访问注意事项**：

```c
// 在大端和小端系统之间传输数据时需要转换
uint32_t read_word_little_endian(uint8_t *addr) {
    return (uint32_t)addr[0] |
           ((uint32_t)addr[1] << 8) |
           ((uint32_t)addr[2] << 16) |
           ((uint32_t)addr[3] << 24);
}

uint32_t read_word_big_endian(uint8_t *addr) {
    return ((uint32_t)addr[0] << 24) |
           ((uint32_t)addr[1] << 16) |
           ((uint32_t)addr[2] << 8) |
           (uint32_t)addr[3];
}
```

## 存储容量与编址

Nor Flash 的容量标识和地址编址是硬件设计和软件开发的重要基础。理解容量与地址位宽的关系，有助于正确配置存储管理单元和地址映射。

### 容量标识方法

Nor Flash 的容量通常使用两种方式标识：比特（bit）和字节（Byte）。

**容量单位**：

| 单位 | 符号 | 含义 |
|------|------|------|
| Kbit | Kb | 1024 bit |
| Mbit | Mb | 1024 Kbit = 1,048,576 bit |
| Gbit | Gb | 1024 Mbit = 1,073,741,824 bit |
| Byte | B | 8 bit |
| KB | KB | 1024 Byte |
| MB | MB | 1024 KB |

**容量换算示例**：

```
128Mbit = 128 × 1024 × 1024 bit = 16,777,216 bit
         = 16,777,216 / 8 Byte = 2,097,152 Byte = 2 MB

256Mbit = 256 × 1024 × 1024 bit = 32,768,256 bit
         = 32,768,256 / 8 Byte = 4,096,512 Byte = 4 MB

512Mbit = 512 × 1024 × 1024 bit = 67,108,864 bit
         = 67,108,864 / 8 Byte = 8,388,608 Byte = 8 MB
```

**芯片命名中的容量标识**：

主流厂商的命名规则通常包含容量信息：

```
Winbond W25Q128:
- "128" 表示 128Mbit = 16MB

Macronix MX25L25645G:
- "256" 表示 256Mbit = 32MB

GigaDevice GD25Q512MC:
- "512" 表示 512Mbit = 64MB
```

### 地址位宽与容量关系

地址位宽决定了芯片可寻址的最大容量范围。

**地址位宽与容量对照表**：

| 地址位数 | 可寻址单元数 | 最大容量 (Byte) | 最大容量 (bit) |
|----------|--------------|-----------------|----------------|
| 20-bit | 1,048,576 | 1 MB | 8 Mbit |
| 21-bit | 2,097,152 | 2 MB | 16 Mbit |
| 22-bit | 4,194,304 | 4 MB | 32 Mbit |
| 23-bit | 8,388,608 | 8 MB | 64 Mbit |
| 24-bit | 16,777,216 | 16 MB | 128 Mbit |
| 25-bit | 33,554,432 | 32 MB | 256 Mbit |
| 26-bit | 67,108,864 | 64 MB | 512 Mbit |
| 27-bit | 134,217,728 | 128 MB | 1 Gbit |
| 28-bit | 268,435,456 | 256 MB | 2 Gbit |

**计算公式**：

```
容量 (Byte) = 2^(地址位数)
最大地址 = 容量 - 1

例如 24 位地址：
容量 = 2^24 Byte = 16,777,216 Byte = 16 MB
最大地址 = 0xFFFFFF (24位全1)
```

**SPI Nor Flash 地址需求**：

```
容量     地址位数    地址字节    说明
─────────────────────────────────────────
8Mbit    23-bit      3 Byte       需要 23 位地址
16Mbit   24-bit      3 Byte       需要 24 位地址
32Mbit   25-bit      3/4 Byte     需要 4 Byte 地址模式
64Mbit   26-bit      4 Byte       需要 4 Byte 地址模式
128Mbit  27-bit      4 Byte       需要 4 Byte 地址模式
256Mbit  28-bit      4 Byte       需要 4 Byte 地址模式
```

### 实际可用容量 vs 标称容量

Nor Flash 芯片的标称容量与实际可用容量之间可能存在差异。

**标称容量**：

- 芯片设计和宣传的容量
- 通常以 bit 为单位标注
- 是 2 的幂次方（如 128Mbit、256Mbit）

**实际可用容量**：

- 扣除以下部分后的可用空间：
  - 特殊功能区域（OTP、安全寄存器等）
  - 扇区间隙（部分非均匀扇区芯片）
  - 坏扇区（出厂坏块）
  - 预留区域

**常见容量差异**：

```
标称 16MB (128Mbit) 芯片实际可用空间：
- W25Q128JV: 16,777,216 Byte (完全可用)
- MX25L12845G: 16,777,316 Byte (略多，预留区域)

标称 32MB (256Mbit) 芯片实际可用空间：
- W25Q256JV: 33,554,432 Byte (完全可用)
- GD25Q256C: 33,554,432 Byte (完全可用)
```

**地址边界与容量计算**：

```
W25Q128 (16MB, 24-bit 地址)：
- 起始地址：0x000000
- 结束地址：0xFFFFFF
- 可用地址范围：0x000000 - 0xFFFFFF
- 地址总数：16,777,216 = 2^24

W25Q256 (32MB, 需要 4-byte 地址模式)：
- 起始地址：0x00000000
- 结束地址：0x01FFFFFF
- 可用地址范围：0x00000000 - 0x01FFFFFF
- 地址总数：33,554,432 = 2^25
```

## 特殊区域

Nor Flash 芯片通常包含一些特殊功能的存储区域，这些区域用于启动加载、参数存储、安全保护等特定用途。

### 引导扇区（Boot Sector）

引导扇区是专门用于存储启动代码的特殊扇区，通常具有更小的扇区粒度和更高的可靠性。

**引导扇区的特点**：

- 位于芯片起始地址（0x000000）
- 通常采用较小的扇区大小（4KB）
- 可配置为只读或读写
- 支持启动代码的快速访问和更新

**典型引导扇区布局**：

```
┌─────────────────────────────────────────────────────────┐
│ 0x000000 - 0x000FFF │ Boot Sector 0 (4KB) - 启动代码    │
├─────────────────────┤
│ 0x001000 - 0x001FFF │ Boot Sector 1 (4KB)               │
├─────────────────────┤
│ 0x002000 - 0x002FFF │ Boot Sector 2 (4KB)               │
├─────────────────────┤
│ 0x003000 - 0x003FFF │ Boot Sector 3 (4KB) - 备份/参数   │
├─────────────────────┤
│ 0x004000 - 0x00FFFF │ Extended Boot (32KB)              │
├─────────────────────┤
│ 0x010000 - xxxxxxxx │ Main Application Area             │
└─────────────────────────────────────────────────────────┘
```

**引导扇区的应用**：

- 存储 MCU/FPGA 的启动代码
- 存放 bootloader
- 保存关键配置参数
- 实现安全启动验证

### 参数存储区（Parameter Area）

参数存储区是专门用于存储配置参数、校准数据等小量数据的特殊区域。

**参数存储区的特点**：

- 采用小扇区结构（通常 4KB）
- 独立于主存储区域
- 可多次编程（相比主存储区更适合频繁更新）
- 部分芯片提供硬件保护

**参数存储区示例**：

```
┌─────────────────────────────────────────────────────────┐
│                    Main Storage                         │
│              (Program/Filesystem Area)                  │
├─────────────────────────────────────────────────────────┤
│ 0x0X0000 - 0x0X0FFF │ Parameter Sector 0 (4KB)          │
├─────────────────────┤
│ 0x0X1000 - 0X1FFF   │ Parameter Sector 1 (4KB)          │
├─────────────────────┤
│ 0x0X2000 - 0X2FFF   │ Parameter Sector 2 (4KB) - 冗余   │
├─────────────────────┤
│ 0x0X3000 - 0X3FFF   │ Parameter Sector 3 (4KB) - 锁定   │
└─────────────────────────────────────────────────────────┘
```

**参数存储区的典型用途**：

- 设备序列号和 MAC 地址
- 校准参数和配置数据
- 用户设置和偏好
- 固件版本信息

### OTP（One-Time Programmable）区域

OTP 区域是一种只能编程一次的存储区域，用于存储不可修改的永久性数据。

**OTP 区域的特点**：

- 编程后无法擦除或修改
- 容量较小（通常 1KB - 64KB）
- 提供硬件级保护机制
- 适合存储唯一标识符和安全密钥

**OTP 区域的结构**：

```
┌─────────────────────────────────────────────────────────┐
│                    Main Storage                          │
│                    (可擦除区域)                          │
├─────────────────────────────────────────────────────────┤
│ OTP Base Address: 0x1FF000                               │
│ ┌─────────────────┬────────────────────────────────────┐ │
│ │ OTP Lock Byte  │ OTP Data Area                      │ │
│ │ (锁定控制)      │ (一次性可编程区域)                   │ │
│ └─────────────────┴────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**OTP 区域的典型应用**：

- 芯片序列号（唯一 ID）
- 加密密钥
- 产品认证信息
- 版权保护数据

**OTP 编程示例**：

```c
// OTP 区域编程流程（以 Winbond W25Qxx 为例）
void otp_program(uint32_t offset, uint8_t *data, uint32_t len) {
    // 1. 发送 OTP 编程命令
    write_enable();

    // 2. 写入 OTP 数据
    send_command(0x42);  // OTP Program command
    send_address(otp_base + offset);
    send_data(data, len);

    // 3. 等待编程完成
    wait_busy();

    // 4. 锁定 OTP（可选，锁定后不可再编程）
    // write_otp_lock();
}
```

### 安全寄存器（Security Registers）

安全寄存器是提供额外安全保护的特殊存储区域，通常用于存储加密密钥、认证数据等敏感信息。

**安全寄存器的特点**：

- 独立于主存储区域
- 可设置读写密码保护
- 支持一次性锁定
- 部分芯片提供硬件加密引擎接口

**安全寄存器的类型**：

| 类型 | 说明 | 容量 |
|------|------|------|
| 一次性可编程锁定 | 编程后可永久锁定 | 256-4KB |
| 可重复编程安全区 | 可多次擦写，需密码访问 | 1KB-16KB |
| 加密存储区 | 硬件级加密保护 | 依芯片而定 |

**安全寄存器结构**：

```
┌─────────────────────────────────────────────────────────┐
│                    Main Storage                          │
│                    (用户存储区)                          │
├─────────────────────────────────────────────────────────┤
│ Security Registers:                                      │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ Security Register 0 (256B) - 序列号                 │ │
│ ├─────────────────────────────────────────────────────┤ │
│ │ Security Register 1 (256B) - 密钥存储              │ │
│ ├─────────────────────────────────────────────────────┤ │
│ │ Security Register 2 (256B) - 认证数据              │ │
│ └─────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────┤
│ OTP Area (64KB) - 一次性编程区域                         │
└─────────────────────────────────────────────────────────┘
```

**安全寄存器的应用场景**：

- 设备唯一标识符（UID）
- 公钥/私钥存储
- 数字签名数据
- 防克隆认证信息

**访问控制机制**：

```
安全级别：
┌─────────────────────────────────────────────────────────┐
│ Level 0: 完全开放                                       │
│          - 主存储区完全可访问                            │
├─────────────────────────────────────────────────────────┤
│ Level 1: 读保护                                         │
│          - 安全寄存器需要密码读取                        │
├─────────────────────────────────────────────────────────┤
│ Level 2: 写保护                                         │
│          - 安全寄存器锁定后不可修改                       │
├─────────────────────────────────────────────────────────┤
│ Level 3: 完全锁定                                       │
│          - 安全寄存器永久锁定，无法访问                   │
└─────────────────────────────────────────────────────────┘
```

## 本章小结

本章详细介绍了 Nor Flash 的存储结构与寻址机制，主要包括以下内容：

1. **芯片内部组织结构**：
   - 存储阵列由浮栅晶体管组成矩阵结构
   - 扇区是擦除操作的最小单位
   - 页是编程操作的约束单位
   - 支持字节、字、双字等多种访问宽度

2. **扇区/块结构类型**：
   - 均匀扇区：所有扇区大小相同，管理简单
   - 非均匀扇区：包含多种规格（4KB+32KB+64KB），适合复杂应用
   - 主流芯片采用不同的扇区布局设计

3. **地址映射机制**：
   - SPI Nor Flash 使用 3 字节或 4 字节地址
   - 字节寻址是最常见的访问方式
   - 需注意地址对齐要求
   - 大多数芯片采用小端模式

4. **存储容量与编址**：
   - 容量以 bit 或 Byte 标注
   - 地址位宽决定最大可寻址容量
   - 实际可用容量可能略大于或小于标称值

5. **特殊区域**：
   - 引导扇区用于启动代码存储
   - 参数存储区适合频繁更新的小数据
   - OTP 区域存储一次性编程的永久数据
   - 安全寄存器提供硬件级安全保护

理解这些存储结构与寻址机制，将为后续的 Nor Flash 驱动开发打下坚实基础。

# SPI 接口详解

SPI（Serial Peripheral Interface）是一种同步串行通信协议，广泛应用于嵌入式系统中主控芯片与外设之间的数据传输。作为 Nor Flash 最常用的接口之一，深入理解 SPI 协议对于掌握 Nor Flash 驱动开发至关重要。本章将从硬件层面详细讲解 SPI 总线的工作原理、数据传输模式、时序要求以及在 Nor Flash 中的具体应用。

---

## SPI 总线基础

### SPI 协议概述

SPI 是由 Motorola 在上世纪 80 年代初提出的一种全双工、同步串行通信协议。与 I2C 等其他串行协议相比，SPI 以其简单性和高速传输能力著称，特别适用于对数据传输速率有较高要求的场景。SPI 协议最初设计用于微控制器与外围芯片之间的短距离通信，如今已广泛应用于 Nor Flash、传感器、显示器、无线模块等多种外设的接口中。

SPI 协议的核心优势包括：全双工通信能力允许数据同时收发；简单的硬件接口降低了设计复杂度；高速传输能力可达数百 MHz 时钟频率；支持多从设备灵活扩展。这些特点使 SPI 成为 Nor Flash 存储芯片的首选接口，尤其是随着移动设备和物联网设备的快速发展，SPI 接口的 Nor Flash 因其小封装、低功耗和低成本而获得广泛应用。

从协议分层角度看，SPI 处于 OSI 模型的物理层和数据链路层之间，它定义了物理电气特性（电压等级、时序要求）和数据传输格式（字节顺序、时钟关系），但不包含寻址机制和传输确认。SPI 协议的简单性既是优势也是限制：优势在于实现成本低、功耗低、速度快；限制在于没有硬件层面的流控制和错误检测机制，需要在应用层自行实现。

### 四线制接口

SPI 接口采用四线制设计，每根信号线都有明确的功能定义，共同构成完整的通信通道。理解这四根信号线的功能是掌握 SPI 协议的基础。

**时钟线（SCK / SCLK）**

时钟线是 SPI 通信的基准，由主设备产生并提供给从设备。时钟信号决定了数据传输的时序，所有数据的采样和输出都以时钟边沿为基准。SCK 的频率决定了 SPI 接口的传输速率，常见频率包括 10MHz、50MHz、100MHz、133MHz 等，高速应用可达 200MHz 甚至更高。时钟频率的选择需要考虑从设备的最高支持频率以及 PCB 布线的信号完整性。

在空闲状态下，SCK 可以保持在高电平或低电平，具体由时钟极性（CPOL）决定。时钟信号的占空比通常为 50%，但某些高速应用可能会调整占空比以优化信号质量。时钟线的驱动能力需要根据连接从设备的数量和布线长度进行评估，必要时可在主设备端添加缓冲器。

**主机输出从机输入线（MOSI / DI / SI）**

MOSI（Master Out Slave In）是主设备向从设备传输数据的主数据线。在 SPI 通信中，主设备通过 MOSI 将数据发送到从设备。数据在时钟的上升沿或下降沿由主设备输出，从设备在该边沿后的某个时刻进行采样。MOSI 的方向始终是从主到从，即使在全双工模式下，数据的流向也不会改变。

MOSI 线的电气特性需要与从设备匹配，包括信号电压等级、输入门限等。对于高速传输，MOSI 线应按照传输线理论进行设计，必要时添加串联电阻以抑制信号过冲和振铃。当连接多个从设备时，MOSI 信号通常采用星型或总线型拓扑结构，需要注意信号负载效应。

**主机输入从机输出线（MISO / DO / SO）**

MISO（Master In Slave Out）是从设备向主设备传输数据的反向数据线。在 SPI 通信中，从设备通过 MISO 将数据发送回主设备。与 MOSI 类似，数据在时钟的上升沿或下降沿由从设备输出，主设备在该边沿进行采样。MISO 线在从设备未被选中时通常处于高阻态，以避免总线冲突。

在标准 SPI 配置中，MISO 仅在从设备被选中（CS 信号有效）时才驱动信号线，未选中时呈高阻态。这一特性允许多个从设备共享同一 MISO 总线而不会产生冲突。设计时需要确认从设备的这一行为，部分芯片可能默认输出，需要通过软件配置或外部电路来实现高阻态控制。

**片选线（CS / SS / CE）**

片选线（Chip Select）用于激活特定的从设备，使主设备能够独立访问总线上的多个从设备而不产生冲突。当 CS 信号为有效电平（通常为低电平）时，对应的从设备被选中并参与 SPI 通信；CS 为无效电平时，从设备忽略总线上的所有信号，其 MISO 输出呈高阻态。

SPI 接口的片选信号可以是低电平有效或高电平有效，具体取决于从设备的设计。大多数 Nor Flash 芯片采用低电平有效的片选信号。片选信号的时序要求非常严格：必须在时钟开始前建立，并在时钟结束后才能释放。片选信号的边沿应与时钟边沿保持足够的间隔，以确保从设备正确识别选通状态。

### 主从设备角色

SPI 通信采用主从架构，总线上只能有一个主设备，但可以支持多个从设备。主设备负责产生时钟信号和控制片选信号，是通信的发起者；从设备被动响应主设备的请求，在被片选激活后参与数据交换。

**主设备职责**

主设备在 SPI 通信中扮演核心角色，承担以下关键职责。首先是时钟管理：主设备产生并提供 SCK 时钟信号，决定通信的时序和数据传输速率。时钟的极性和相位也由主设备控制，这决定了数据采样的边沿和有效窗口。其次是传输控制：主设备发起每一次数据传输，包括发送命令、地址和数据；同时接收从设备的响应数据。主设备还负责片选管理，通过控制 CS 信号选择目标从设备并协调多从设备访问。

从硬件角度看，SPI 主设备需要具有可配置的时钟发生器、多条独立的 GPIO 用于 MOSI、MISO 和 CS 信号控制。现代 MCU 通常内置 SPI 控制器，可自动处理数据传输的时序细节，简化软件开发的复杂度。

**从设备职责**

从设备被动响应主设备的控制，在被片选激活后参与通信。从设备的核心职责包括：监听 SPI 总线并在被片选时接收命令和地址；执行相应的操作（如读取数据、写入数据、擦除等）；在需要时通过 MISO 线向主设备返回数据或状态。

从设备的 MISO 输出是关键的时序要素。在未被片选时，从设备的 MISO 必须呈高阻态，以避免总线冲突；被片选后，从设备需要在适当的时刻驱动 MISO 线，在时钟边沿后输出数据。从设备的这些行为由其内部状态机控制，软件需要严格按照芯片手册的时序要求进行操作。

### 时钟极性和时钟相位

时钟极性（CPOL，Clock Polarity）和时钟相位（CPHA，Clock Phase）是 SPI 协议中两个关键的可配置参数，它们共同决定了时钟的有效边沿和数据采样的时机。正确理解这两个参数对于调试 SPI 通信问题至关重要。

**时钟极性（CPOL）**

时钟极性定义了 SPI 总线在空闲状态下的时钟电平。当 CPOL = 0 时，空闲状态的 SCK 为低电平，第一个时钟边沿为上升沿；当 CPOL = 1 时，空闲状态的 SCK 为高电平，第一个时钟边沿为下降沿。CPOL 的选择会影响信号的边沿特性和 EMI 性能，需要根据从设备的要求和系统设计进行权衡。

```
CPOL = 0 时的时钟波形：
SCK:  ________|--------|--------|--------|--------
              ^        ^        ^        ^        ^
           空闲    第一个上升沿  上升沿   上升沿   上升沿

CPOL = 1 时的时钟波形：
SCK:  ________|--------|--------|--------|--------
              ^        ^        ^        ^        ^
           空闲    第一个下降沿  下降沿   下降沿   下降沿
```

**时钟相位（CPHA）**

时钟相位定义了数据采样的时钟边沿和数据的移出时机。当 CPHA = 0 时，数据在时钟的第一个边沿被采样（如果 CPOL = 0 则为上升沿，CPOL = 1 则为下降沿）；当 CPHA = 1 时，数据在时钟的第二个边沿被采样。CPHA 的选择决定了数据建立和保持的时间窗口。

```
CPHA = 0 时的数据采样时机：
SCK:     ____      ____      ____      ____
         ^  ^    ^  ^    ^  ^    ^  ^
       边沿1   边沿2   边沿3   边沿4
              |
           数据采样点

CPHA = 1 时的数据采样时机：
SCK:     ____      ____      ____      ____
         ^  ^    ^  ^    ^  ^    ^  ^
       边沿1   边沿2   边沿3   边沿4
              |
           数据采样点
```

### SPI 工作模式

将 CPOL 和 CPHA 两个参数组合，可以得到四种 SPI 工作模式（Mode 0-3）。这四种模式在数据采样时机和数据输出时机上有所不同，开发者必须确保主从设备使用相同的工作模式才能正确通信。

**Mode 0（CPOL = 0，CPHA = 0）**

Mode 0 是最常用的 SPI 模式，也是许多 Nor Flash 芯片的默认工作模式。在 Mode 0 中，空闲时钟为低电平，数据在上升沿采样，下降沿输出。这意味着每个时钟周期包含一个下降沿（用于数据输出）和一个上升沿（用于数据采样）。

```
Mode 0 时序图（CPOL=0, CPHA=0）：

        Bit 7        Bit 6        Bit 5        Bit 0
        <--- 8 clocks ---><--- 8 clocks ---><--- 8 clocks --->
SCK:    ___|---|___|---|___|---|___|---|___|---|___|---|___
            ^       ^       ^       ^       ^
          输出    采样    输出    采样    输出
MOSI:   Z   D7  D6  D5  D4  D3  D2  D1  D0   Z
            ^               ^               ^
         数据建立     数据建立     数据建立
            |---------------|---------------|
                    采样窗口
```

**Mode 1（CPOL = 0，CPHA = 1）**

Mode 1 与 Mode 0 类似，空闲时钟为低电平，但数据采样时机不同。在 Mode 1 中，数据在下降沿输出，上升沿采样。这提供了不同的建立和保持时间，适用于特定从设备的要求。

**Mode 2（CPOL = 1，CPHA = 0）**

Mode 2 的空闲时钟为高电平，数据在下降沿采样，上升沿输出。Mode 2 的信号特性与 Mode 0 相反，某些老旧或特殊设计的从设备可能需要此模式。

**Mode 3（CPOL = 1，CPHA = 1）**

Mode 3 的空闲时钟为高电平，数据在上升沿输出，下降沿采样。这是 Mode 1 的镜像配置，同样适用于需要特定时序的从设备。

```
四种 SPI 模式对比：

        Mode 0          Mode 1          Mode 2          Mode 3
        (CPOL=0,        (CPOL=0,        (CPOL=1,        (CPOL=1,
         CPHA=0)         CPHA=1)         CPHA=0)         CPHA=1)
       ─────────────────────────────────────────────────────────
空闲SCK   低电平          低电平          高电平          高电平
采样边沿  上升沿          下降沿          下降沿          上升沿
输出边沿  下降沿          上升沿          上升沿          下降沿
```

对于大多数 Nor Flash 芯片，Mode 0 是最常用的工作模式。在实际应用中，开发者应始终参考芯片数据手册中规定的时序要求，配置主设备的 SPI 控制器以匹配从设备的工作模式。

## SPI 数据传输模式

随着技术的发展，SPI 接口从最初的 Single SPI 逐步演进到 Dual SPI、Quad SPI、Octal SPI 以及 DDR 模式，每种模式都在数据传输速率上有显著提升。理解这些传输模式的特点和适用场景，对于选择合适的 Nor Flash 和优化系统性能至关重要。

### Single SPI（单线模式）

Single SPI 是最原始的 SPI 传输模式，也称为标准 SPI 或传统 SPI。在 Single SPI 模式下，数据的发送和接收分别使用独立的信号线（MOSI 和 MISO），实现全双工通信。这意味着在同一时钟周期内，主设备可以同时发送和接收数据，充分利用了 SPI 的同步特性。

**工作原理**

Single SPI 的数据传输以字节为单位进行。在每个时钟周期内，主设备通过 MOSI 线发送一位数据，同时从设备通过 MISO 线返回一位数据。经过 8 个时钟周期后，一个字节的数据传输完成。这种通信方式简洁高效，适用于大多数基本的数据传输场景。

```
Single SPI 全双工传输时序：

        发送字节 0xA5 (10100101)          接收字节 0x5A (01011010)
SCK:    ___|---|___|---|___|---|___|---|___|---|___|---|___|---|___
MOSI:   Z   1   0   1   0   0   1   0   1   Z
            D7  D6  D5  D4  D3  D2  D1  D0
MISO:   Z   0   1   0   1   1   0   1   0   Z
            D7  D6  D5  D4  D3  D2  D1  D0

        <--- 8 bits --->                <--- 8 bits --->
        发送 MSB→LSB                    接收 MSB→LSB
```

**性能特点**

Single SPI 的理论带宽计算公式为：带宽 = 时钟频率 × 1 位/时钟周期。以常见的 100MHz 时钟频率为例，Single SPI 的理论带宽为 100Mbps（约 12.5MB/s）。考虑到协议开销（如命令、地址传输），实际可用带宽通常为理论值的 60-80%。

Single SPI 模式的优势在于其简单性和广泛的兼容性，几乎所有的 SPI 设备都支持此模式。其缺点是数据传输效率较低，在需要高速数据传输的应用中可能成为瓶颈。对于读取性能要求不高的场景，Single SPI 仍然是经济实惠的选择。

### Dual SPI（双线模式）

Dual SPI 是 SPI 接口向高速传输演进的第一个重要里程碑。通过将数据线的宽度从 1 位扩展到 2 位，Dual SPI 在不增加时钟频率的情况下将数据传输速率提升了一倍。这种模式牺牲了全双工特性，改为半双工通信，但在数据吞吐量上的优势使其成为许多高性能 Nor Flash 的标准配置。

**工作原理**

在 Dual SPI 模式下，MOSI 和 MISO 两根数据线被重新定义为 IO0 和 IO1，用于双向数据传输。命令和地址仍然通过 IO0 单线传输，但数据 payload 可以通过 IO0 和 IO1 同时传输，每时钟周期传输 2 位数据。这种模式通常被称为 1-2-2 模式（1 条命令线、2 条地址线、2 条数据线）。

```
Dual SPI 模式数据传输：

        发送命令和地址              读取数据（双线）
IO0:    C   A7  A6  A5  A4  A3  A2  A1  A0  D7  D5  D3  D1  D9  D7  D5
IO1:    Z   A7  A6  A5  A4  A3  A2  A1  A0  D6  D4  D2  D0  D8  D6  D4
             ^               ^               ^               ^
          命令(1bit)      地址(2bit)      数据(2bit)     数据(2bit)

        每2位占用1个时钟周期，8位数据仅需4个时钟周期
```

**性能提升**

相比 Single SPI，Dual SPI 的数据传输效率提升约 100%。以 100MHz 时钟为例，Dual SPI 的理论带宽为 200Mbps（约 25MB/s），实际可用带宽约为 15-20MB/s。这一提升对于需要快速读取固件或配置数据的应用非常有价值。

Dual SPI 模式的典型应用包括：固件代码的快速读取（XIP 执行）、配置数据的加载、大容量数据的预读取等。许多现代 MCU 的 SPI 控制器内置 Dual SPI 模式支持，可以无缝切换。

**命令序列**

进入 Dual SPI 模式通常需要通过特定的命令。以 Winbond W25QxxJV 系列为例，发送命令 0x3B（Dual Read Data）后，地址和数据传输都自动切换为双线模式。软件只需在初始化时选择合适的读取命令即可启用 Dual SPI 模式。

### Quad SPI（四线模式）

Quad SPI 将数据线的宽度进一步扩展到 4 位，使数据传输速率达到 Single SPI 的四倍。这是目前最广泛使用的高速 SPI 模式，尤其在嵌入式系统的代码存储和 XIP 执行场景中应用普遍。Quad SPI 在保持合理引脚数的同时，提供了接近并行接口的性能。

**工作原理**

Quad SPI 使用 4 根数据线（IO0-IO3）进行数据传输。其中 IO0 和 IO1 复用 MOSI 和 MISO 功能，IO2 复用 /WP（写保护）引脚，IO3 复用 /HOLD 引脚。这种引脚复用设计使得 Quad SPI 可以在标准的 8 引脚封装上实现，无需增加额外的引脚。

```
Quad SPI 模式引脚分配：

    Single SPI          Quad SPI
    ─────────────────────────────
    MOSI    (IO0)       IO0 (双向数据)
    MISO    (IO1)       IO1 (双向数据)
    /WP     (IO2)       IO2 (双向数据 / /WP)
    /HOLD   (IO3)       IO3 (双向数据 / /HOLD)

    数据传输采用 1-4-4 模式：
    1 条命令线 → 4 条地址线 → 4 条数据线
```

在 Quad SPI 模式下，命令仍然通过 IO0 单线发送，地址通过 IO0-IO3 四线发送，数据也通过 IO0-IO3 四线传输。这使得每时钟周期可以传输 4 位数据，8 位数据仅需 2 个时钟周期。

```
Quad SPI 读取时序：

        命令           地址                   数据
IO0:    C   A7  A6  A5  A4  D7  D3  D11 D7  D3  D11 D7  D3
IO1:    Z   A7  A6  A5  A4  D6  D2  D10 D6  D2  D10 D6  D2
IO2:    Z   A7  A6  A5  A4  D5  D1  D9  D5  D1  D9  D5  D1
IO3:    Z   A7  A6  A5  A4  D4  D0  D8  D4  D0  D8  D4  D0

        <1-bit><-------- 4-bit --------><---------- 16-bit ----------->

        命令    地址(每线1bit)         数据(每线4bit，共4线)
        1 CLK   4 CLK                  4 CLK (16 bits)
```

**性能特点**

Quad SPI 在 133MHz 时钟下的理论带宽为 532Mbps（约 66.6MB/s），实际可用带宽约为 50-60MB/s。这一性能水平足以支持大多数嵌入式系统的代码存储和直接执行（XIP）需求。相比传统的并行 Nor Flash，Quad SPI 以更少的引脚数实现了接近的读取性能。

Quad SPI 模式的启用需要通过命令切换。以常见的 0x6B（Quad Read Data）命令为例，发送该命令后，SPI 控制器自动进入 Quad 模式进行后续数据传输。部分芯片还支持在初始化时配置默认的 Quad 模式，简化软件操作。

### Octal SPI（八线模式）

Octal SPI 是 SPI 接口的最新演进，使用 8 根数据线进行数据传输，提供了单线模式的八倍带宽。Octal SPI 主要面向高性能应用，如高端嵌入式系统、汽车电子和 AI 边缘计算设备，这些应用需要更快的启动时间和代码执行速度。

**工作原理**

Octal SPI 将数据线扩展到 8 根（IO0-IO7），每时钟周期可传输 8 位数据。这种模式通常采用 1-8-8 命令序列（1 条命令线、8 条地址线、8 条数据线），或者在某些实现中使用 8-8-8 序列（命令也通过 8 线传输）。

```
Octal SPI 模式数据传输：

        发送命令 (1-bit via IO0)  地址 (8-bit via IO0-IO7)  数据 (8-bit x 8 lines)
IO0:    C    A7  A6  A5  A4  A3  A2  A1  A0  D7  D15 D23 D31 D39 D47 D55
IO1:    Z    A7  A6  A5  A4  A3  A2  A1  A0  D6  D14 D22 D30 D38 D46 D54
IO2:    Z    A7  A6  A5  A4  A3  A2  A1  A0  D5  D13 D21 D29 D37 D45 D53
IO3:    Z    A7  A6  A5  A4  A3  A2  A1  A0  D4  D12 D20 D28 D36 D44 D52
IO4:    Z    A7  A6  A5  A4  A3  A2  A1  A0  D3  D11 D19 D27 D35 D43 D51
IO5:    Z    A7  A6  A5  A4  A3  A2  A1  A0  D2  D10 D18 D26 D34 D42 D50
IO6:    Z    A7  A6  A5  A4  A3  A2  A1  A0  D1  D9  D17 D25 D33 D41 D49
IO7:    Z    A7  A6  A5  A4  A3  A2  A1  A0  D0  D8  D16 D24 D32 D40 D48

        1 CLK   1 CLK                    1 CLK                    1 CLK
        (CMD)   (每线1bit)               (每线1bit)               (每线8bit)
```

**性能特点**

Octal SPI 在 200MHz 时钟下的理论带宽为 1.6Gbps（约 200MB/s），实际可用带宽约为 150-180MB/s。更重要的是，Octal SPI 通常与 DDR（双倍数据速率）技术结合使用，在时钟的上升沿和下降沿都进行数据传输，使有效带宽翻倍至 3.2Gbps（约 400MB/s）。

Octal SPI 的信号完整性设计更具挑战性，8 根数据线需要严格匹配长度并正确端接。此外，Octal SPI 需要主控芯片的硬件支持，不是所有 MCU 都具备 Octal SPI 控制器，通常需要使用专门的 SPI Flash 控制器或 FPGA 实现。

### DDR（双倍数据速率）模式

DDR（Double Data Rate）模式通过在时钟的上升沿和下降沿都进行数据传输，进一步提升 SPI 接口的带宽。DDR 技术最早应用于同步 DRAM，如今已成为高速 SPI 接口的标准配置，特别是在 Octal SPI 和 XPI（eXtended SPI）接口中。

**工作原理**

在标准 SPI 模式下，数据仅在时钟的上升沿或下降沿（取决于工作模式）进行传输。而在 DDR 模式下，数据在时钟的上升沿和下降沿都进行传输，每时钟周期可传输 2 位数据。结合多线传输模式（如 Quad DDR、Octal DDR），DDR 可以实现极高的数据传输速率。

```
DDR 模式与 SDR 模式对比（以 Quad SPI 为例）：

SDR Quad SPI（每时钟周期 4 位）：
SCK:    ^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^
        ┌───┐       ┌───┐       ┌───┐       ┌───┐
IO:     │ D3│  D2   │ D1│  D0   │ D7│  D6   │ D5│  D4
            └───┘       └───┘       └───┘       └───
        <--1 CLK-->   <--1 CLK-->   <--1 CLK-->   <--1 CLK-->
        (4 bits)     (4 bits)     (4 bits)     (4 bits)

DDR Quad SPI（每时钟周期 8 位）：
SCK:    ^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^|^^^
        ┌─┬─┐     ┌─┬─┐     ┌─┬─┐     ┌─┬─┐
IO:     │D3│D2│ D1 │D0│ D7  │D6│ D5   │D4│
        └─┴─┘     └─┴─┘     └─┴─┘     └─┴─┘
        <--1 CLK-->   <--1 CLK-->   <--1 CLK-->   <--1 CLK-->
        (8 bits)     (8 bits)     (8 bits)     (8 bits)
```

**性能对比**

DDR 模式相比 SDR 模式可实现约 100% 的带宽提升。配合 Octal SPI，DDR 模式可实现最高 400MB/s 以上的读取速度，接近传统并行 Nor Flash 的性能水平。这一性能对于需要快速启动或高速代码执行的系统尤为重要。

```
各模式理论带宽对比（以 133MHz 时钟为例）：

模式              数据位宽    边沿采样    有效带宽
─────────────────────────────────────────────────────────
Single SPI SDR    1-bit      单边沿      ~16 MB/s
Dual SPI SDR      2-bit      单边沿      ~33 MB/s
Quad SPI SDR      4-bit      单边沿      ~66 MB/s
Octal SPI SDR     8-bit      单边沿      ~133 MB/s
Quad SPI DDR      4-bit      双边沿      ~133 MB/s
Octal SPI DDR     8-bit      双边沿      ~266 MB/s

注：实际带宽受协议开销、芯片性能等因素影响，上述数值为理论参考值
```

## SPI 时序详解

理解 SPI 时序是进行 Nor Flash 驱动开发的基础。SPI 协议虽然简单，但时序要求严格，任何偏差都可能导致数据传输错误。本节将详细分析 SPI 通信的时序参数，包括建立时间、保持时间、时钟边沿等关键要素，并提供时序图解帮助读者建立直观理解。

### 完整时序图解

SPI 通信的完整时序包括片选建立、命令发送、地址发送、数据传输和片选释放等阶段。每个阶段都有严格的时序要求，需要主设备严格按照数据手册的规范进行配置。

```
标准 SPI 读取操作完整时序图：

        CS#     ___________                         ___________
                |         |                         |
        ________|         |_________________________|__________
                      ↑
                  片选有效

        SCK     ____|---|___|---|___|---|___|---|___|---|___|---|____
                    ┌───┐     ┌───┐     ┌───┐     ┌───┐
                    │   │     │   │     │   │     │   │
        ___________|   |_____|   |_____|   |_____|   |___________
                    ↑       ↑       ↑       ↑
                空闲    第一个   上升沿   上升沿
                        边沿

        MOSI    Z   C   A7  A6  A5  A4  A3  A2  A1  A0  X   X   X
                ↑   ↑                   ↑       ↑
             片选  命令                地址    虚拟
             有效 建立                 建立    时钟

        MISO    Z   X   X   X   X   X   X   X   X   X   D7  D6  D5
                                            ↑       ↑
                                         数据建立  数据采样
                                            |-------|
                                                ↑
                                           数据有效窗口


时间轴说明：
─────────────────────────────────────────────────────────────
tCSS    片选建立时间（CS下降沿到第一个SCK边沿）
tCSH    片选保持时间（最后一个SCK边沿到CS上升沿）
tSCLK   时钟周期
tCLKH   时钟高电平宽度
tCLKL   时钟低电平宽度
tDS     数据建立时间
tDH     数据保持时间
```

### 时序参数说明

SPI 时序参数定义了信号边沿之间的时间关系，是确保可靠通信的关键。不同厂商的 Nor Flash 芯片对这些参数有不同的要求，开发者需要参考具体芯片的数据手册进行配置。

**时钟参数**

时钟参数定义了 SCK 信号的特性，包括时钟周期、时钟高/低电平宽度等。这些参数直接影响数据传输速率和信号的边沿质量。

```
tSCLK    时钟周期（Clock Period）
         定义：相邻两个时钟边沿之间的时间间隔
         计算：tSCLK = 1 / fSCLK
         示例：100MHz 时钟 → tSCLK = 10ns

tCLKH    时钟高电平宽度（Clock High Time）
         定义：时钟信号保持高电平的最短时间
         要求：通常 ≥ 0.4 × tSCLK

tCLKL    时钟低电平宽度（Clock Low Time）
         定义：时钟信号保持低电平的最短时间
         要求：通常 ≥ 0.4 × tSCLK

时钟参数示意图：

SCK:    ┌─────────┐         ┌─────────┐
        │         │         │         │
        │  tCLKH  │  tCLKL  │  tCLKH  │
        └─────────┴─────────┴─────────┘
                    ↑
               tSCLK = tCLKH + tCLKL
```

**建立时间和保持时间**

建立时间（Setup Time）和保持时间（Hold Time）是数字系统中两个最基本的时序参数，它们定义了数据相对于时钟边沿的有效窗口。

```
tSU     数据建立时间（Data Setup Time）
        定义：在时钟采样边沿之前，数据必须保持稳定的最短时间
        测量：从数据信号稳定到采样边沿的时间间隔

tH      数据保持时间（Data Hold Time）
        定义：在时钟采样边沿之后，数据必须保持稳定的最短时间
        测量：从采样边沿到数据开始变化的时间间隔

建立时间和保持时间示意图：

        上升沿采样（CPHA=0）
        ─────────────────────────────────────────
SCK:            ┌─────────┐
                ↑         ↓
                │ 上升沿  │
                └─────────┘
                    ↑
                采样点

        ────────────────────────────────
        │<- tSU ->|
        ────────────|          |────────
        数据         |          |
        有效区       |          |
                    |<-  tH  ->|
                              ─────────
                              数据变化

        有效数据窗口 = tSU + tH
```

**片选时序参数**

片选信号的时序要求确保从设备正确识别选通信号，并在适当的时机参与通信。

```
tCSS    片选建立时间（Chip Select Setup Time）
        定义：从 CS 信号有效（低电平）到第一个时钟边沿的时间
        要求：通常 ≥ 2ns，具体取决于芯片规格

tCSH    片选保持时间（Chip Select Hold Time）
        定义：从最后一个时钟边沿到 CS 信号无效（高电平）的时间
        要求：通常 ≥ 5ns，具体取决于芯片规格

片选时序示意图：

CS#     ────|                    |────────────────────
              ↑ tCSS           ↑ tCSH
              |                 |
        第一个时钟边沿    最后一个时钟边沿
```

### 上升沿/下降沿采样

SPI 协议的灵活性体现在数据采样边沿的选择上，开发者可以根据从设备的要求选择合适的采样边沿。理解不同采样模式的时序特性，有助于优化系统设计和排查通信问题。

**上升沿采样 vs 下降沿采样**

在 Mode 0（CPOL=0）中，数据在上升沿被采样，这意味着数据应该在上升沿之前已经稳定。如果数据在上升沿时刻发生变化，可能导致采样错误。

```
上升沿采样时序（Mode 0）：

SCK:         ___|---|___|---|___|---|___
                 ↑       ↑       ↑
              上升沿  上升沿  上升沿

MOSI:    ────D7───────D6───────D5───────D4──
              ↑               ↑
           建立完成        采样点
           (tSU后)        (上升沿)

MISO:    ────D7───────D6───────D5───────D4──
              ↑               ↑
           建立完成        采样点
           (tSU后)        (上升沿)
```

在 Mode 1（CPHA=1）中，数据在下降沿被采样，提供不同的时序安排。这种模式可能在某些系统中提供更好的信号质量，特别是在高速传输时。

```
下降沿采样时序（Mode 1）：

SCK:         ___|---|___|---|___|---|___
                 ↓       ↓       ↓
              下降沿  下降沿  下降沿

MOSI:    ────D7───────D6───────D5───────D4──
                    ↑               ↑
                 采样点        采样点
                 (下降沿)      (下降沿)
```

**时序裕量分析**

时序裕量是设计中需要重点关注的指标。足够的时序裕量可以确保系统在器件参数变化、温度变化和电压波动等条件下仍然可靠工作。

```
时序裕量计算示意图：

        数据有效窗口              实际采样窗口
        <────────────>            <────────────>
        |←───── tSU ───→|                    |←──── tH ────→|
        |               |                    |              |
        |    理论窗口   |                    |   实际窗口  |
        |               |                    |              |
    ────┼───────────────┼────────────────────┼──────────────┼────
        ↑               ↑                    ↑              ↑
     数据建立        采样边沿             采样边沿      数据变化
     时刻           (理论)              (实际)

    时序裕量 = 实际时序余量 - 芯片要求
            = (tSU_actual + tH_actual) - (tSU_min + tH_min)

    正裕量：系统可正常工作
    负裕量：系统可能发生数据传输错误
```

**时序问题排查**

当 SPI 通信出现错误时，时序问题是常见的原因之一。以下是一些典型的时序问题及其排查方法。

时钟频率过高可能导致数据采样不稳定，特别是在信号完整性较差的情况下。解决方法包括降低时钟频率、增加信号线的串联电阻、改善 PCB 布局等。片选时序不满足要求可能导致从设备响应错误，需要确保 CS 信号在时钟开始前有足够的建立时间，并在时钟结束后有足够的保持时间。数据边沿质量问题可能导致采样错误，可以通过调整驱动强度、添加端接电阻或降低时钟频率来改善。

## Nor Flash SPI 命令

Nor Flash 芯片通过 SPI 命令进行访问，每种命令都有特定的功能和时序要求。理解这些命令是编写 Nor Flash 驱动的基础。本节将详细介绍 Nor Flash 中常用的 SPI 命令，包括读取、写入、擦除和状态寄存器操作等命令的格式和用途。

### 命令格式概述

Nor Flash 的 SPI 命令采用固定的格式序列，通常包括命令码、地址和数据三个部分。不同类型的操作使用不同的命令序列，理解这些格式对于正确驱动 Nor Flash 至关重要。

**标准命令序列**

典型的 Nor Flash SPI 命令序列包含以下几个阶段：首先是命令码阶段，发送 8 位命令码定义要执行的操作；然后是地址阶段，发送 24 位地址（3 字节）指定操作的目标位置；接下来是虚拟时钟阶段（部分命令需要），用于为从设备准备数据输出提供内部处理时间；最后是数据阶段，传输实际的数据 payload。

```
标准 SPI 命令序列结构：

        阶段          时钟数     数据方向      说明
        ────────────────────────────────────────────────────
        命令码         8        MOSI→Flash    操作类型
        地址          24        MOSI→Flash    目标地址
        虚拟时钟      8/16      -             内部处理时间
        数据          N×8       MOSI→Flash    写入数据
                                  或
                                 MISO→Host    读取数据

        实际序列示例（读取命令 0x03）：
        ────────────────────────────────────────────────────
        8 clocks  │ 24 clocks │ 8 clocks │ N×8 clocks
           ↓           ↓           ↓           ↓
        ┌─────┬─────────┬─────────┬─────────────────┐
        │ CMD │ ADDR[23:0]│ dummy  │   DATA[7:0]     │
        │ 0x03│  0xXX    │  0x00   │   读取的数据    │
        └─────┴─────────┴─────────┴─────────────────┘
```

**命令编码约定**

Nor Flash 的命令编码遵循一定的约定，了解这些约定有助于理解不同命令的设计逻辑。位操作命令通常使用特定的位域编码，普通命令使用 0x00-0xFF 之间的值。读取命令通常以 0x0 或 0x1 开头，写入命令通常以 0x0 或 0x2 开头，擦除命令通常以 0x2 或 0x5 开头，寄存器操作命令通常以 0x0、0x5 或 0x3 开头。这些约定并非强制，但多数厂商遵循这一模式。

### 常用读取命令

读取命令用于从 Nor Flash 中获取存储的数据。根据数据传输模式和速度的不同，Nor Flash 支持多种读取命令。以下是常见的读取命令及其特点。

**0x03 - Read Data（标准读取）**

0x03 是最基本的读取命令，在 Single SPI 模式下工作。该命令的特点是兼容性好，几乎所有 SPI Nor Flash 都支持此命令，但其速度相对较慢，不支持高速传输模式。

```
命令 0x03 详细时序：

        1. 发送命令码 0x03
        2. 发送 24 位地址（A23-A0）
        3. 读取数据（每字节 8 时钟）

SCK:    ___|---|___|---|___|---|___|---|___|---|___|---|___|---|___|---|____
MOSI:   Z   0   0   0   0   0   1   1   A23 A22 ... A1  A0
            CMD=0x03              地址字节

MISO:   Z   X   X   X   X   X   X   X   X   X   X   X   X   D7  D6 ...
                                    地址周期    第一个数据字节
```

**0x0B - Fast Read（快速读取）**

0x0B 是标准快速读取命令，与 0x03 相比增加了 8 位虚拟时钟周期，用于为从设备提供额外的内部准备时间。虽然速度提升有限，但该命令是后续高速模式的基础。

**0x3B - Dual Read（双线读取）**

0x3B 是 Dual SPI 模式的读取命令，地址和数据都通过双线传输，提供约 2 倍于标准读取的速度提升。该命令需要 Flash 芯片支持 Dual SPI 模式。

**0x6B - Quad Read（四线读取）**

0x6B 是 Quad SPI 模式的读取命令，数据通过四线传输，提供约 4 倍于标准读取的速度提升。这是目前最常用的快速读取命令，适用于支持 Quad SPI 的芯片。

**0xEB - Octal/High Speed Read（八线/高速读取）**

0xEB 是针对高速传输优化的读取命令，支持 Quad SPI 或 Octal SPI 模式。该命令通常需要更少的虚拟时钟周期，并且支持 DDR 模式以实现更高的数据传输速率。

```
常见读取命令对比表：

命令码    名称                    数据线    时钟要求      典型速度(@133MHz)
─────────────────────────────────────────────────────────────────────────
0x03     Read Data              Single     标准          ~16 MB/s
0x0B     Fast Read              Single     8 dummy       ~16 MB/s
0x3B     Dual Read              Dual       8 dummy       ~33 MB/s
0x6B     Quad Read              Quad       8 dummy       ~66 MB/s
0xEB     Octal/High Speed Read  Quad+      16/32 dummy   ~133-266 MB/s

注：dummy 时钟数量因芯片和模式而异，需参考具体数据手册
```

### 常用写入命令

写入命令用于将数据写入 Nor Flash。与读取操作不同，写入操作前需要确保目标扇区已被擦除，并且通常需要检查 Flash 的写入状态。

**0x02 - Page Program（页编程）**

0x02 是最常用的写入命令，用于在已擦除的 Flash 空间中写入数据。该命令支持写入 1-256 字节（一个页大小），但不能跨页边界写入。

```
命令 0x02 时序图：

SCK:    ___|---|___|---|___|---|___|---|___|---|___|---|___|---|___|---|____
MOSI:   Z   0   0   0   0   0   1   0   A23 A22 ... A1  A0  D7  D6 ... D0
            CMD=0x02              地址字节           数据字节

MISO:   Z   X   X   X   X   X   X   X   X   X   X   X   X   X   X   X
                                                    (忽略)

写入流程：
1. 发送页编程命令 0x02
2. 发送 24 位起始地址
3. 发送要写入的数据（1-256 字节）
4. 等待写入完成（轮询状态寄存器或等待 /BUSY 引脚）
```

**0x32 - Quad Input Page Program（四线页编程）**

0x32 是 Quad SPI 模式的页编程命令，地址和数据都通过四线传输，显著缩短写入时间。该命令需要芯片支持 Quad SPI 模式。

**0x38 - Extended Quad Input Page Program（扩展四线页编程）**

0x38 是另一种四线页编程命令，与 0x32 相比具有更灵活的地址配置选项，适用于某些特定的应用场景。

```
写入命令对比表：

命令码    名称                           数据线    写入大小     典型速度
─────────────────────────────────────────────────────────────────────────
0x02     Page Program                  Single    1-256B      ~2 MB/s
0x32     Quad Input Page Program       Quad      1-256B      ~8 MB/s
0x38     Extended Quad Input PP        Quad      1-256B      ~8 MB/s

注意事项：
- 写入前必须确保目标区域已擦除
- 不能跨页边界写入
- 写入操作期间需轮询状态位或等待 /BUSY 信号
```

### 常用擦除命令

擦除是 Flash 存储器的特有操作，它将存储单元恢复为擦除状态（逻辑1），为后续写入操作做准备。Nor Flash 支持多种粒度的擦除命令。

**0x20 - Sector Erase（扇区擦除）**

0x20 是最小粒度的擦除命令，用于擦除一个 4KB 扇区。扇区擦除是 Nor Flash 最常用的擦除方式，因为它提供了最小的擦除粒度，有助于减少不必要的写入量。

```
命令 0x20 时序：

SCK:    ___|---|___|---|___|---|___|---|___|---|___|---|___|---|___
MOSI:   Z   0   0   1   0   0   0   0   A23 A22 ... A1  A0
            CMD=0x20              扇区地址

        扇区地址：通常是 4KB 对齐的地址
        例如：0x000000, 0x001000, 0x002000 等

擦除时间：典型值 30-200ms
```

**0x52 - Block Erase（块擦除）**

0x52 用于擦除一个 32KB 或 64KB 块，比扇区擦除的粒度更大。块擦除的速度比扇区擦除快，适用于需要快速擦除大量数据的场景。

**0xD8 - Block Erase（64KB 块擦除）**

0xD8 是另一种块擦除命令，通常用于擦除 64KB 的块。这是大容量芯片中常用的擦除方式，在擦除速度和粒度之间取得平衡。

**0xC7 / 0x60 - Chip Erase（整片擦除）**

0xC7 和 0x60 都是整片擦除命令，用于一次性擦除整个 Flash 芯片。整片擦除时间较长（通常需要数十秒），但可以快速清除所有数据。

```
擦除命令对比表：

命令码    名称                   粒度       典型擦除时间     适用场景
─────────────────────────────────────────────────────────────────────────
0x20     Sector Erase           4KB        30-200ms       精细数据管理
0x52     Block Erase            32KB       100-400ms      批量数据更新
0xD8     Block Erase            64KB       200-800ms      批量数据更新
0x60     Chip Erase             整个芯片   10-100s        完全重置
0xC7     Chip Erase             整个芯片   10-100s        完全重置
```

### 状态寄存器操作命令

Nor Flash 内置状态寄存器，用于反映芯片的当前状态，包括写入使能、写入正在进行、错误标志等信息。正确使用状态寄存器是实现可靠 Flash 操作的关键。

**0x05 - Read Status Register-1（读取状态寄存器1）**

0x05 用于读取状态寄存器 1，该寄存器包含最常用的状态位，如写入使能锁（BUSY/WEL）和写保护控制位。

```
状态寄存器 1 位定义：

位     名称         描述
────────────────────────────────────────
S0     BUSY        写入操作进行中标志
                    1 = 正在进行
                    0 = 空闲

S1     WEL         写入使能锁
                    1 = 写入使能
                    0 = 写入禁止

S2-S3  BP[1:0]     块保护位
                    决定哪些区域被保护

S4     TB          顶部/底部块保护

S5     SEC         扇区/块保护粒度

S6     SPRL        状态寄存器保护锁

S7     SRP0       状态寄存器保护位

命令时序：
SCK:    ___|---|___|---|___|---|___|---|___|---|___|---|___|---|___
MOSI:   Z   0   0   0   0   0   1   0   X   X   X   X   X   X   X   X
            CMD=0x05

MISO:   Z   X   X   X   X   X   X   X   S7  S6  S5  S4  S3  S2  S1  S0
                                        状态寄存器数据
```

**0x06 - Write Enable（写入使能）**

0x06 用于设置写入使能锁（WEL）位，允许后续的写入或擦除操作。在执行任何写入或擦除命令之前，必须先发送 0x06 命令使能写入。

**0x04 - Write Disable（写入禁止）**

0x04 用于清除写入使能锁（WEL）位，禁止后续的写入或擦除操作。在完成写入或擦除操作后，通常需要发送此命令以防止意外写入。

**0x01 - Write Status Register（写入状态寄存器）**

0x01 用于写入状态寄存器，可以一次性配置多个状态位，包括写保护设置、块保护等。

```
状态寄存器操作命令表：

命令码    名称                    功能
────────────────────────────────────────────
0x05     Read SR1               读取状态寄存器 1
0x35     Read SR2               读取状态寄存器 2
0x15     Read SR3               读取状态寄存器 3
0x01     Write SR1/2/3          写入状态寄存器
0x06     Write Enable           设置 WEL 位
0x04     Write Disable          清除 WEL 位
```

## 时序参数配置

正确配置 SPI 控制器的时序参数是确保 Nor Flash 可靠工作的关键。本节将讨论时钟频率选择、工作模式配置以及时序裕量分析等实践问题，帮助开发者优化 SPI 接口的性能和稳定性。

### 时钟频率选择

选择合适的 SPI 时钟频率需要综合考虑多个因素，包括 Flash 芯片的规格限制、主控芯片的能力、PCB 信号质量以及系统噪声环境等。

**Flash 芯片规格限制**

每款 Nor Flash 芯片都有其支持的最高时钟频率，这由芯片内部的物理限制决定。低成本芯片通常支持 50-100MHz，高性能芯片可达 133MHz、166MHz 甚至 200MHz。超过芯片支持的最大频率会导致数据采样错误。

```
常见 Nor Flash 芯片的时钟频率支持：

芯片型号              容量      Single SPI   Quad SPI   Octal SPI
────────────────────────────────────────────────────────────────
W25Q80DV             8Mbit     104MHz       80MHz      N/A
W25Q128JV            128Mbit   133MHz       133MHz     N/A
W25Q256JV            256Mbit   133MHz       133MHz     200MHz
W25Q512JVQ           512Mbit   166MHz       133MHz     200MHz
MX25L25645G          256Mbit   133MHz       133MHz     N/A
MX25UM51245G         512Mbit   166MHz       133MHz     200MHz
GD25Q256C            256Mbit   120MHz       120MHz     N/A

注：具体规格请参考各芯片数据手册
```

**主控芯片能力**

主控芯片的 SPI 控制器能力也是重要考虑因素。不同的 MCU 提供不同频率范围和模式的 SPI 控制器。一些低端 MCU 可能仅支持最高 10-20MHz 的时钟频率，而高性能 MCU 可支持 100MHz 以上的 SPI 时钟。

此外，硬件控制器可能对特定模式有支持限制。例如，某些 SPI 控制器可能不支持 Quad SPI 或 Octal SPI，需要通过软件模拟或外加芯片实现。开发前应仔细评估主控芯片的能力。

**信号完整性考虑**

随着时钟频率的提高，信号完整性问题变得越来越突出。高频信号在 PCB 走线上会出现衰减、反射和串扰等问题。走线长度不匹配会导致信号歪斜，阻抗不连续会导致反射振铃。

```
PCB 设计建议：

1. 走线长度匹配
   - SCK 与数据线长度差控制在 50mil (1.27mm) 以内
   - CS 信号长度可以稍长，但不宜超过 200mil

2. 阻抗控制
   - 使用 50Ω 阻抗走线
   - 保持走线下方完整的地平面
   - 避免跨越分割平面

3. 端接电阻
   - 在信号源端串联 22-33Ω 电阻
   - 可减少过冲和振铃

4. 布局要求
   - Flash 芯片靠近主控放置
   - 减少走线长度（建议 < 3 英寸）
   - 远离干扰源
```

### 模式选择建议

根据不同的应用场景，应选择合适的 SPI 工作模式以平衡性能和兼容性。

**模式选择决策树**

选择 SPI 模式时应考虑以下因素。首先考虑速度需求：对于低速或简单应用，Single SPI 兼容性好、实现简单；对于需要快速启动或 XIP 执行的场景，选择 Quad SPI 或 Octal SPI。其次考虑主控能力：确认主控芯片是否支持目标模式，部分低成本 MCU 可能不支持高速模式。然后考虑 Flash 芯片支持：检查所选 Flash 芯片是否支持所需的传输模式。最后考虑代码复杂度：更高的模式通常需要更复杂的驱动代码和更严格的时序控制。

```
模式选择建议表：

应用场景                    推荐模式        典型速度      复杂度
────────────────────────────────────────────────────────────────────
简单配置存储               Single SPI     ~10 MB/s     低
一般代码存储                Single SPI     ~16 MB/s     低
快速启动 (bootloader)       Quad SPI       ~50 MB/s     中
XIP 代码执行                Quad SPI DDR   ~80 MB/s     中
高性能应用                  Octal SPI      ~200 MB/s    高
极致性能                    Octal SPI DDR  ~400 MB/s     非常高
```

### 时序裕量分析

时序裕量是评估设计可靠性的重要指标。足够的时序裕量可以保证系统在器件差异、温度变化和电压波动等条件下稳定工作。

**裕量计算方法**

时序裕量通过比较实际时序和芯片要求的时序来计算。实际时序由主控芯片的 SPI 控制器和 PCB 设计决定，而芯片要求则来自 Flash 器件的数据手册。

```
裕量计算示例：

假设条件：
- SPI 时钟频率：100MHz (tSCLK = 10ns)
- Flash 芯片要求：tSU ≥ 3ns, tH ≥ 3ns
- 主控实际输出：tSU_actual = 5ns, tH_actual = 5ns

裕量计算：
- 建立时间裕量 = tSU_actual - tSU_required = 5ns - 3ns = 2ns (正裕量)
- 保持时间裕量 = tH_actual - tH_required = 5ns - 3ns = 2ns (正裕量)

结论：设计满足时序要求，有 2ns 的裕量
```

**裕量不足的解决方法**

如果时序裕量为负值（即不满足要求），可以采取以下措施进行调整。降低时钟频率是最直接的方法，减少时钟周期可以增加数据有效窗口。优化 PCB 布局可以减少信号延迟和歪斜，提高信号质量。调整驱动强度可以改变信号的上升/下降时间，改善建立和保持时间。添加缓冲器可以改善信号完整性和驱动能力。

```
裕量优化建议：

问题                  原因              解决方案
─────────────────────────────────────────────────────────────────
建立时间不足          时钟太快/走线太长  降低频率/缩短走线
保持时间不足          数据线延迟太大    调整走线长度匹配
采样窗口过窄          信号边沿太慢      使用更快的驱动器
信号过冲/振铃         阻抗不匹配        添加串联电阻
```

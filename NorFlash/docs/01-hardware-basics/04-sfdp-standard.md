# SFDP 串行接口标准详解

SFDP（Serial Flash Discoverable Parameters，串行闪存可发现参数）是 JEDEC（JEDEC Solid State Technology Association，电子器件工程联合会）制定的一种标准机制，用于让 SPI Nor Flash 能够自描述其硬件参数和功能特性。通过 SFDP，主控芯片可以在不预先了解 Flash 芯片具体型号的情况下，自动识别芯片的各项参数，从而实现驱动程序的通用化设计。本章将深入讲解 SFDP 标准的各个方面，帮助读者全面理解这一重要规范。

---

## SFDP 标准概述

### SFDP 定义与背景

SFDP 是 JEDEC 为串行闪存设备制定的一种标准参数查询机制，其核心思想是让 Flash 芯片主动“告诉”主控芯片自己的各项参数，而不是要求主控预先存储所有芯片的型号信息。这一机制类似于 PCI 设备的 PCI ID 自动识别，或者 USB 设备的设备描述符，极大地简化了嵌入式系统的 Flash 驱动开发。

在 SFDP 出现之前，驱动工程师必须为每种 Flash 芯片型号编写专门的驱动代码或配置数据表。这不仅增加了软件维护的复杂度，也限制了系统对不同 Flash 芯片的兼容性。当需要更换 Flash 供应商或使用不同容量芯片时，往往需要修改驱动程序。SFDP 的出现彻底改变了这一局面，使得“一种驱动，多种芯片”成为可能。

SFDP 机制的实现依赖于 Flash 芯片内部一段专用的参数存储区域，这段区域按照 JEDEC 规定的格式存储各种参数信息。主控芯片通过发送特定的 SFDP 读取命令（5Ah）来访问这段参数区，获取芯片的各项规格信息。这些信息包括芯片容量、擦除块大小、支持的传输模式、时序参数等，为驱动代码的配置提供了所有必要的依据。

### JEDEC 标准体系

SFDP 规范是 JEDEC 标准体系中的重要组成部分，经过多年发展已形成多个版本。理解这些版本之间的差异和演进关系，对于开发兼容多种芯片的驱动代码非常重要。

**JESD216（原始版本）**

JESD216 是 SFDP 标准的原始版本，于 2011 年首次发布。该版本定义了 SFDP 的基本数据结构，包括 SFDP 头部和基础参数表（BFPT，Basic Flash Parameter Table）。JESD216 规定了参数存储的基本格式，支持 3 字节地址模式，是后续版本发展的基础。虽然是最初版本，但 JESD216 仍被许多早期芯片支持。

**JESD216A（第一修订版）**

JESD216A 在原始版本基础上进行了扩展，增加了对 4 字节地址模式的支持。随着大容量 Flash 芯片的出现（超过 128Mb），3 字节地址已经无法寻址全部存储空间。JESD216A 引入了 4 字节地址参数，使主控能够识别芯片是否支持 4 字节地址模式，以及如何切换到该模式。这一版本还增加了更多时序参数的描述。

**JESD216B（第二修订版）**

JESD216B 进一步扩展了参数表达能力，增加了扇区映射表（Sector Map Table）的支持。不同厂商的 Flash 芯片在扇区结构上存在差异，有些支持 4KB 扇区，有些只有 64KB 块。JESD216B 引入了统一的扇区描述机制，使主控能够准确了解芯片的扇区布局。此外，该版本还增加了对更多传输模式和厂商特定参数的支持。

**JESD216D.01（当前最新版本）**

JESD216D 是 SFDP 标准的最新版本（.01 是勘误版本），在 JESD216B 基础上进行了多项重要更新。该版本增加了对 Octal SPI 传输模式的支持，以满足高性能应用的需求。同时，JESD216D 还引入了更详细的时序参数描述、复兴（Resume）命令支持、以及更灵活的厂商参数扩展机制。目前主流的 Flash 芯片大多支持 JESD216D 标准。

```
SFDP 标准版本演进：

版本        发布年份    主要新增功能
─────────────────────────────────────────────────────────────
JESD216    2011        初始版本，基本参数表 BFPT
JESD216A   2013        4字节地址模式支持
JESD216B   2015        扇区映射表，多传输模式
JESD216D   2019        Octal SPI，更详细时序，扩展参数
```

### 版本演进与兼容性

理解 SFDP 版本之间的兼容性对于编写稳健的驱动代码非常重要。JEDEC 在设计 SFDP 时考虑了向后兼容性，使得支持新版本标准的驱动也能与旧版芯片正常工作。

SFDP 头部的版本号字段（Major Version 和 Minor Version）用于标识参数表的格式版本。驱动代码在读取 SFDP 时，首先读取头部获取版本信息，然后根据版本号选择相应的解析逻辑。较新版本的头部可以包含指向旧版参数表的指针，实现向后兼容。

在参数解析方面，驱动代码应该采用“按需读取”的策略。当需要某个参数时，首先检查当前版本的参数表中是否存在该参数，如果不存在则使用默认值或尝试从厂商特定参数中获取。这种设计可以确保驱动在不同版本的芯片上都能正常工作，只是某些高级功能可能在旧芯片上不可用。

### SFDP 的优势

SFDP 标准为嵌入式系统开发带来了显著的优势，这些优势使其成为现代 Nor Flash 驱动的标准配置。

**自动识别能力**

SFDP 最直接的优势是实现了 Flash 芯片的自动识别。主控芯片在上电后读取 SFDP 参数，即可获知芯片的各项规格，无需人工配置或预先存储芯片型号信息。这一特性极大地简化了系统启动流程，特别适合需要支持多种 Flash 芯片的平台。自动识别还使得系统能够在运行时检测 Flash 芯片的更换，为现场可更换单元（FRU）等应用提供了支持。

**驱动通用化**

有了 SFDP 参数，单一版本的驱动代码可以支持数十种甚至数百种不同的 Flash 芯片。这不仅减少了驱动的维护工作量，也使得系统能够灵活选择不同供应商的芯片，降低供应链风险。驱动开发者无需为每种芯片单独调试参数，只需按照 SFDP 标准解析参数并配置硬件即可。

**未来兼容性**

SFDP 是一种面向未来的设计。随着 Flash 技术的演进和新的功能特性的出现，JEDEC 会通过更新标准版本来支持这些新特性。支持 SFDP 的驱动代码可以通过解析新版参数表来获取新功能信息，无需修改驱动代码即可支持新芯片。这种设计确保了软件的长期可维护性。

## SFDP 数据结构

SFDP 参数存储在 Flash 芯片内部的一段专用区域中，这段区域按照 JEDEC 规定的结构进行组织。理解 SFDP 的数据结构是正确解析参数的基础。

### SFDP 头部格式

SFDP 头部是访问参数的入口点，它提供了整个参数表的概览信息，包括版本号和参数头的数量。

```
SFDP 头部结构（8 字节）：

字节偏移    字段名称              描述
─────────────────────────────────────────────────────────────
0x00       Signature (31:24)    SFDP 标识符高位（固定值 0x53）
0x01       Signature (23:16)    SFDP 标识符次高位（固定值 0x46）
0x02       Signature (15:8)     SFDP 标识符次低位（固定值 0x44）
0x03       Signature (7:0)      SFDP 标识符低位（固定值 0x50）
                                注：完整签名 = 0x50444653 = "SFDP"
0x04       Minor Revision       SFDP 次版本号（如 0x06 表示 v1.6）
0x05       Major Revision       SFDP 主版本号（如 0x01 表示 v1.x）
0x06       Number of Parameter  参数头数量（从 0 开始计数，
            Headers (N)          实际数量 = N + 1）
0x07       Reserved              保留（固定值 0xFF）
```

SFDP 签名（Signature）字段是验证数据有效性的重要依据。读取 SFDP 时，主控应该首先检查这 4 个字节是否为 0x50、0x46、0x44、0x50（即 ASCII 字符串 "SFDP"）。如果不匹配，说明读取位置错误或 Flash 不支持 SFDP。

版本号字段（Minor Revision 和 Major Revision）定义了参数表的格式版本。驱动代码可以根据版本号选择相应的解析逻辑。例如，版本 1.0（Major=0x01, Minor=0x00）对应原始的 JESD216 版本，而版本 1.6（Major=0x01, Minor=0x06）对应 JESD216A。

```
SFDP 次版本号与 JEDEC 标准对应关系：

Minor Revision    对应 JEDEC 版本
────────────────────────────────────────
0x00              JESD216
0x06              JESD216A
0x07              JESD216B
0x08              JESD216B.01
0x09              JESD216C
0x0A              JESD216D
```

参数头数量字段（Number of Parameter Headers）指明了后续参数头的个数。实际数量等于该字段值加 1，因为参数头从 0 开始编号。大多数芯片至少包含两个参数头：基础参数表（BFFPT）和扇区映射表（Sector Map Table）。

### 参数头（Parameter Header）格式

每个参数表都有一个对应的参数头，用于描述参数表的位置、长度和版本信息。

```
参数头结构（8 字节）：

字节偏移    字段名称              描述
─────────────────────────────────────────────────────────────
0x00       Parameter ID LSB     参数 ID 低字节（JEDEC 分配）
0x01       Parameter ID MSB     参数 ID 高字节
0x02       Parameter Table      参数表主版本号
            Major Revision
0x03       Parameter Table      参数表次版本号
            Minor Revision
0x04       Parameter Table      参数表长度（单位：字节）
            Length (7:0)
0x05       Parameter Table      参数表长度（高位，需配合偏移）
            Length (15:8)
0x06       Parameter Table      参数表地址偏移（从 SFDP 起始位置计算）
            Pointer (7:0)        低字节
0x07       Parameter Table      参数表地址偏移（从 SFDP 起始位置计算）
            Pointer (15:8)       高字节

参数 ID 编码规则（JESD216）：
    0x0000 - SFDP 头部信息
    0x0001 - 基础 Flash 参数表 (BFPT)
    0x0002 - 扇区映射表
    0x0003 - 4 字节地址参数表
    0xFFxx - 厂商特定参数（xx 为厂商代码）
```

参数 ID（Parameter ID）是识别参数表类型的关键标识。JEDEC 为标准参数表分配了固定的 ID 值，而厂商特定参数使用 0xFFxx 格式的 ID，其中 xx 是厂商代码。例如，Winbond 厂商代码为 0x01，因此其特定参数的 ID 为 0xFF01。

参数表指针（Parameter Table Pointer）给出了参数表数据相对于 SFDP 起始地址的偏移量。这个设计允许厂商在芯片内部灵活安排参数存储位置，同时保持访问接口的一致性。驱动代码在解析参数头后，根据指针值计算出参数表的实际地址，然后读取参数数据。

参数表长度（Parameter Table Length）以字节为单位给出了每个参数表的大小。驱动代码需要根据这个长度分配足够的缓冲区来存储参数数据，避免读取溢出。

```
SFDP 数据结构完整视图：

┌─────────────────────────────────────────────────────────────────┐
│                        SFDP 头部 (8 字节)                        │
│  签名: 0x50444653  │ 版本: 1.6  │ 参数头数: 2                   │
└─────────────────────────────────────────────────────────────────┘
                                │
        ┌───────────────────────┼───────────────────────┐
        ▼                       ▼                       ▼
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│  参数头 0     │     │  参数头 1     │     │  参数头 N     │
│  ID: 0x0001   │     │  ID: 0x0002   │     │  ID: 0xFFxx   │
│  → BFPT       │     │  → 扇区映射表 │     │  → 厂商参数   │
└───────────────┘     └───────────────┘     └───────────────┘
        │                       │                       │
        ▼                       ▼                       ▼
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│  BFPT 数据    │     │  扇区映射数据 │     │  厂商参数数据 │
│  (16-32 字节) │     │  (可变长度)   │     │  (可变长度)   │
└───────────────┘     └───────────────┘     └───────────────┘
```

## JEDEC 基础参数表（BFPT）详解

基础参数表（BFPT，Basic Flash Parameter Table）是 SFDP 中最重要的参数表，它包含了 Flash 芯片的核心规格信息。BFPT 由 JEDEC 标准定义，所有支持 SFDP 的芯片都必须实现该表。

### BFPT 整体结构

BFPT 是一个结构化的参数表，定义了芯片的容量、擦除块大小、支持的传输模式、时序参数等核心信息。BFPT 的长度在 JESD216 原始版本中为 16 字节，在后续版本中扩展到 32 字节。

```
BFPT 字节定义总览（JESD216D，32 字节）：

字节偏移    内容概述                    字节偏移    内容概述
────────────────────────────────────────────────────────────────
0x00-0x01   块/扇区擦除配置            0x10-0x11   Quad Enable 信息
0x02-0x03   架构/组织信息               0x12-0x13   页面编程配置
0x04-0x05   芯片容量（密度）            0x14-0x15   保留
0x06        芯片容量编码                 0x16        固件/内核版本
0x07        保留                        0x17        保留
0x08-0x09   写入粒度/脉冲宽度           0x18-0x1F   保留（4字节地址模式扩展）
```

### BFPT 详解

以下是 BFPT 中各字节的详细含义和编码规则。

```
BFPT 字节 0-1：块/扇区擦除配置

字节 0:
  [7:4] 保留
  [3:0] 4KB 擦除命令支持
         0000 = 不支持 4KB 擦除
         0001 = 支持命令 DAh（4KB 扇区擦除）
         其他 = 保留

字节 1:
  [7:0] 块/扇区擦除粒度类型
         00h = 块擦除（不支持 4KB 扇区擦除）
         01h = 扇区擦除（专用 4KB 扇区结构）
         其他 = 保留
```

```
BFPT 字节 2-3：架构/组织信息

字节 2:
  [7:6] 保留
  [5]   连续读取模式支持
         0 = 不支持
         1 = 支持
  [4]   页面编程位数
         0 = 8 位（标准）
         1 = 16 位
  [3:2] 保留
  [1:0] 地址模式
         00 = 仅 3 字节地址
         01 = 仅 4 字节地址
         1X = 支持 3/4 字节地址切换

字节 3:
  [7:4] 双倍数据速率（DDR）支持
         0000 = 不支持 DDR
         0001 = 支持 DDR
  [3:0] 数据位宽（IO 数量）
         0000 = x1/x2/x4（仅标准模式）
         0001 = x1/x2/x4/x8（支持 Octal）
         其他 = 保留
```

```
BFPT 字节 4-5：芯片容量（密度）

字节 4: 容量位 [31:24]
字节 5: 容量位 [23:16]

容量编码规则（密度 = 容量位 + 1）：
  容量位 = 容量值 - 1

容量值与实际容量对应关系：
  容量值     实际容量         容量值     实际容量
  ─────────────────────────────────────────────────────
  0x08      8Mbit            0x10      256Mbit
  0x09      16Mbit           0x11      512Mbit
  0x0A      32Mbit           0x12      1Gbit
  0x0B      64Mbit           0x13      2Gbit
  0x0C      128Mbit          ...

计算公式：密度 = (容量位 + 1) bits
  例如：容量位 = 0x17 (23) → 密度 = 24 位 = 16Mbit
```

BFPT 字节 4-5 共同定义了 Flash 芯片的存储容量。JEDEC 采用了一种特殊的编码方式：容量值等于实际容量位宽减 1。例如，8Mbit 芯片的容量值为 7（0x07），128Mbit 芯片的容量值为 127（0x7F）。这种编码方式允许表示从 1Mbit 到 2Gbit 的各种容量。

```
BFPT 字节 6：芯片容量编码（续）

字节 6:
  [7:0] 芯片容量 MSB（容量位 [15:8]）

完整容量计算示例：
  字节 4 = 0x00 (密度位 [7:0])
  字节 5 = 0x10 (密度位 [15:8])
  字节 6 = 0x00 (密度位 [23:16])

  密度值 = 0x000010 (little-endian: 0x0010 << 8 | 0x00)
         = 0x1000 (4096)
  实际容量 = 4097 Mbits = 约 512MB

  注意：具体计算需参考 JEDEC 标准文档
```

```
BFPT 字节 8-9：写入粒度与字节编程时间

字节 8 - 写入粒度（Write Granularity）：
  [7:0] 写入粒度标识
         01h = 1 字节粒度
         02h = 2 字节粒度
         04h = 4 字节粒度
         08h = 8 字节粒度
         10h = 16 字节粒度
         20h = 32 字节粒度
         40h = 64 字节粒度
         80h = 128 字节粒度（页面大小）
         其他 = 保留

字节 9 - 字节编程时间（Byte Program Time）：
  [7:4] 字节编程乘数（Multipler）
        编程时间 = 基础时间 × (乘数值 + 1)
  [3:0] 字节编程基础时间单位
        0000 = 1μs
        0001 = 8μs
        0010 = 16μs
        其他 = 保留
```

```
BFPT 字节 10-11：擦除时间参数

字节 10:
  [7:0] 4KB 扇区擦除时间（典型值，单位：ms）
        0 = 不支持 4KB 擦除
        其他 = 实际时间 = 表中值 × 单位时间

字节 11:
  [7:0] 块擦除时间（典型值，单位：ms）
        计算方式同上
```

```
BFPT 字节 12-13：芯片最大时序

字节 12 - 最大时钟频率（Single/Dual SPI）：
  [7:5] 保留
  [4:0] 时钟频率代码
        频率表：
        01h = 20MHz    02h = 25MHz    03h = 33MHz
        04h = 40MHz    05h = 50MHz    06h = 66MHz
        07h = 80MHz    08h = 100MHz   09h = 104MHz
        0Ah = 110MHz   0Bh = 120MHz   0Ch = 133MHz
        0Dh = 166MHz   0Eh = 200MHz

字节 13 - 最大时钟频率（Quad SPI）：
  编码方式同字节 12
```

```
BFPT 字节 14-15：时序参数

字节 14:
  [7:4] tCLH - 时钟高电平最小时间（Clock High Time）
  [3:0] tCLL - 时钟低电平最小时间（Clock Low Time）

字节 15:
  [7:4] tDS - 数据建立时间（Data Setup Time）
  [3:0] tDH - 数据保持时间（Data Hold Time）

时序代码到实际时间的转换：
  代码  实际时间
  ─────────────────────
  0     0.1ns (不确定)
  1     1ns
  2     2ns
  3     3ns
  4     4ns
  5     5ns
  6     6ns
  7     7ns
  8     8ns
  9-15  保留
```

```
BFPT 字节 16-17：新型擦除时间

字节 16 - 新型 4KB 擦除时间参数
字节 17 - 新型块擦除时间参数

这些字节提供了更精确的擦除时间参数定义，
支持不同擦除类型的时间规格。
```

### 芯片容量编码详解

芯片容量是 BFPT 中最重要的参数之一，它决定了 Flash 存储空间的大小。JEDEC 采用了一种特殊的两层编码方式来表示各种容量。

**第一层：密度位编码**

密度位（Density Bits）是 BFPT 字节 4-6 中的核心编码字段。它使用线性编码，密度值直接等于实际容量位宽减 1。例如：

- 密度位 = 0x00 → 1Mbit
- 密度位 = 0x07 → 8Mbit
- 密度位 = 0x0F → 16Mbit
- 密度位 = 0x1F → 32Mbit
- 密度位 = 0x3F → 64Mbit

**第二层：容量标识符**

对于常见的容量值，JEDEC 还定义了可选的容量标识符。这些标识符出现在 BFPT 特定位置，用于快速识别常用容量：

```
常见芯片容量的标准标识符：

标识符      典型容量        常用芯片型号
─────────────────────────────────────────────────────
0x14        1Mbit           W25Q10
0x15        2Mbit           W25Q20
0x16        4Mbit           W25Q40
0x17        8Mbit           W25Q80
0x18        16Mbit          W25Q16
0x19        32Mbit          W25Q32
0x1A        64Mbit          W25Q64
0x1B        128Mbit         W25Q128
0x1C        256Mbit         W25Q256
0x1D        512Mbit         W25Q512
0x1E        1Gbit           W25Q01
```

### 擦除块大小

擦除块大小是影响 Flash 使用方式的重要参数。不同容量的芯片和不同厂商的产品可能采用不同的块结构。

**4KB 扇区支持**

BFPT 字节 0 中的 4KB 擦除位（Bit 0）指示芯片是否支持 4KB 扇区擦除命令（0xDA）。支持 4KB 扇区擦除的芯片允许更精细的存储管理，特别适合需要频繁更新少量数据的应用场景。

**块/扇区架构类型**

BFPT 字节 1 中的架构类型位定义了芯片的整体擦除结构：

- 类型 00h：块擦除架构，只有 64KB（或 32KB）块
- 类型 01h：扇区擦除架构，包含专用的 4KB 扇区

某些芯片支持混合架构，既有大块也有 4KB 小扇区，这需要通过扇区映射表（Sector Map Table）来详细描述。

### 支持的传输模式

BFPT 描述了芯片支持的各种数据传输模式，这些信息对于选择合适的读取命令至关重要。

**单线/双线/四线模式**

BFPT 字节 2-3 中的位字段定义了芯片支持的 IO 模式：

- Bit 2-0：数据位宽支持
- Bit 3：16 位模式支持
- Bit 5：连续读取模式

```
传输模式支持标志：

标志              含义
─────────────────────────────────────────────────────
x1 (SIO)         仅 Single SPI 模式
x2 (DIO)         支持 Dual SPI 模式
x4 (QIO)         支持 Quad SPI 模式
x8 (OIO)         支持 Octal SPI 模式

连续读取模式      支持连续地址读取（无需每字节重发命令）
```

**DDR 模式支持**

BFPT 字节 3 中的 DDR 支持位指示芯片是否支持双倍数据速率（DDR）传输。在 DDR 模式下，数据在时钟的上升沿和下降沿都能进行传输，可以将有效带宽翻倍。

### 页面编程大小

页面编程（Page Program）是 Flash 的基本写入操作，页面大小决定了单次写入的最大数据量。

BFPT 字节 8 中的写入粒度字段定义了页面大小。标准 Nor Flash 通常采用 256 字节页面，但某些芯片可能支持更大的页面。写入粒度还影响写入对齐要求，不对齐的写入可能导致数据错误。

## 扇区映射表

扇区映射表（Sector Map Table）是 SFDP 的可选扩展参数，用于描述 Flash 芯片的详细扇区结构。对于具有复杂扇区层次结构的芯片，扇区映射表提供了完整的地层次信息。

### 4KB 扇区参数表

4KB 扇区参数表描述了芯片中 4KB 扇区的配置信息。对于支持 4KB 扇区擦除的芯片，这些参数定义了扇区的具体位置和属性。

```
4KB 扇区参数结构：

字段              字节数    描述
─────────────────────────────────────────────────────
配置标识          1        0x00 = 仅块擦除
                            0x01 = 4KB 扇区优先
                            0x02 = 4KB 扇区仅在参数区
区域描述符长度    1        描述符结构的长度
区域数量          1        连续区域的数量
区域描述符        可变     每个区域的详细描述
```

### 扇区类型描述

扇区映射表将 Flash 存储空间划分为多个区域（Region），每个区域具有相同的擦除特性。描述符中包含每个区域的起始地址、区域大小和擦除类型。

```
区域描述符结构：

字段              字节数    描述
─────────────────────────────────────────────────────
区域类型          1        00h = 4KB 扇区
                            01h = 32KB 块
                            02h = 64KB 块
区域大小          3        区域中的单元数量
                        （与区域类型相乘得到实际大小）
```

### 地址布局信息

完整的扇区映射提供了从芯片基址到末端的完整地址布局，驱动代码可以根据这些信息实现精确的地址管理。

```
扇区映射表示例（W25Q128JV）：

区域    地址范围              大小      擦除类型
─────────────────────────────────────────────────────
0       0x000000 - 0x000FFF   4KB      4KB 扇区擦除
1       0x001000 - 0x01FFFF   32KB     32KB 块擦除
2       0x020000 - 0xFFFFFF   64KB     64KB 块擦除

说明：
- 前 128KB (0x000000 - 0x01FFFF) 可以使用 4KB 或 32KB 擦除
- 其余空间仅支持 64KB 块擦除
```

## SFDP 读取时序

SFDP 读取是主控芯片获取 Flash 参数的基本操作。理解 SFDP 读取的时序要求是正确实现参数读取的关键。

### 读取命令序列

SFDP 读取使用专用的 5Ah 命令，这是 JEDEC 标准定义的 SFDP 读取操作码。

```
SFDP 读取命令序列（5Ah）：

命令格式：
  ┌──────┬────────┬────────┬────────┬────────┬────────┐
  │ CMD  │  ADDR  │ DUMMY  │  DATA  │  DATA  │  ...   │
  │ 0x5A │ 23:16  │  7:0   │ 7:0    │ 7:0    │        │
  │      │ ADDR   │        │        │        │        │
  │ 1    │  1     │   1    │   N    │   N    │        │
  └──────┴────────┴────────┴────────┴────────┴────────┘

  - CMD: 命令码 0x5A
  - ADDR: 24 位地址（从 0x000000 开始）
  - DUMMY: 8 个虚拟时钟周期（部分芯片需要）
  - DATA: 参数数据（LSB 在前）

SFDP 地址空间映射：
  0x000000 - 0x0000FF: SFDP 头部 + 参数头
  0x000030 - 0x00004F: BFPT 数据（16 字节，W25Q128JV）
  0x000050 - 0x00006F: 扇区映射表
  0x000100 - 0x0001FF: 厂商特定参数
```

SFDP 地址采用 24 位地址格式，这与传统的 3 字节地址 Flash 命令一致。地址从 0 开始递增，可以连续读取多个字节的数据。

```
SFDP 读取时序图：

SCK:     ___|---|___|---|___|---|___|---|___|---|___|---|___|---|___|---|___
             ↑       ↑       ↑       ↑       ↑       ↑       ↑
          上升沿  上升沿  上升沿  上升沿  上升沿  上升沿  上升沿

MOSI:    Z   0   1   0   1   1   0   1   0   A7  A6  ... A0  X   X   X   X
             CMD=5Ah                地址字节

MISO:    Z   X   X   X   X   X   X   X   X   X   X   X   X   D7  D6  ... D0  D7
                                    dummy         第一个数据字节  连续读取

时间参数：
  tCSS: CS 建立时间（≥5ns）
  tCSH: CS 保持时间（≥5ns）
  tSCLK: 时钟周期
  tDS: 数据建立时间
  tDH: 数据保持时间
```

### 地址模式切换

部分大容量 Flash 芯片支持 4 字节地址模式，这需要通过 SFDP 参数来识别并正确切换。

**3 字节地址模式**

传统的 SPI Flash 使用 3 字节地址，可以寻址最大 16MB（128Mbit）的存储空间。这种模式兼容性好，是大多数中小容量芯片的默认模式。

**4 字节地址模式**

对于容量超过 128Mbit 的芯片，需要使用 4 字节地址模式来寻址全部存储空间。SFDP 中的 BFPT 参数指示芯片是否支持 4 字节地址模式。

```
4 字节地址模式切换流程：

1. 检查 BFPT 字节 2 的地址模式位
   - 若支持 4 字节地址，进入下一步

2. 发送 4 字节地址进入命令
   - Winbond: 0xB7 (Enter 4-Byte Mode)
   - Micron: 0xB7
   - Macronix: 0xB7

3. 验证地址宽度
   - 尝试访问超出 3 字节范围的地址
   - 确认数据读写正确

4. 退出 4 字节地址模式（如需要）
   - 命令 0xE9 (Exit 4-Byte Mode)

SFDP 参数中的 4 字节地址信息：
  BFPT Offset 0x18-0x1F 包含：
    - 4 字节地址命令支持
    - 4 字节地址读取命令
    - 4 字节地址写入命令
    - 4 字节地址擦除命令
```

```
  - 命令码：0x65（Quad I/O SFDP Read）
  - 地址：4 位/时钟
  - Dummy：4-8 个时钟周期（取决于芯片）
  - 数据：4 位/时钟 × 4 线
```

## 本章小结

本章对 SFDP（Serial Flash Discoverable Parameters）串行接口标准进行了全面深入的讲解，为 Nor Flash 驱动开发提供了重要的理论基础。

在 SFDP 标准概述部分，我们介绍了 SFDP 的定义和背景，它作为 JEDEC 标准的核心作用，以及版本从 JESD216 到 JESD216D 的演进过程。SFDP 的自动识别能力和驱动通用化优势使其成为现代嵌入式系统不可或缺的标准机制。

在 SFDP 数据结构部分，我们详细分析了 SFDP 头部格式（签名、版本号、参数头数量）和参数头格式（参数 ID、版本、长度、指针）。这些结构化数据为驱动代码提供了访问参数表的完整导航信息。

在扇区映射表部分，我们介绍了如何描述复杂的地层次结构，包括 4KB 扇区参数和地址布局信息。这些信息对于实现精细的存储管理至关重要。

在厂商特定参数部分，我们分析了 Winbond、Micron 等主要厂商的扩展参数实现，这些参数提供了标准参数之外的额外功能信息。

在实践案例分析部分，我们通过 Winbond W25Q128JV 和 Micron MT25Q 系列的 SFDP 解析示例，展示了如何将理论付诸实践。同时对比了不同厂商芯片的 SFDP 差异。

通过本章的学习，读者应该能够理解 SFDP 标准的工作原理，掌握 SFDP 数据结构的解析方法，并具备独立实现 SFDP 参数读取的能力。这些知识将直接应用于后续章节的 Nor Flash 驱动开发实践，实现自动识别和通用化驱动的目标。

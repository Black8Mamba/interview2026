# 并行接口详解

并行接口是 Nor Flash 与主控芯片通信的另一种重要方式，相比 SPI 接口具有更高的数据传输带宽和更低的访问延迟。并行接口通过同时使用多根数据线进行数据传输，能够实现高达数百 MB/s 的读取速度，特别适用于对性能要求较高的嵌入式系统、FPGA 配置以及大容量代码存储场景。本章将详细讲解并行 Nor Flash 的接口信号定义、异步时序机制、x8/x16 数据宽度配置、FPGA 专用接口以及 MCU 的 FSMC/FMC 配置方法。

---

## 并行 Nor Flash 概述

### 并行接口与串行接口的对比

并行接口和串行接口代表了两种不同的数据传输方式，各有其适用场景和技术特点。理解两者的差异对于正确选择 Nor Flash 接口类型至关重要。

**并行接口特点**：

并行接口使用多根信号线同时传输数据的每一位。以 x16 模式为例，一次时钟周期可以传输 16 位数据，理论带宽是单线 SPI 的 16 倍（假设相同的工作频率）。这种并行传输方式使并行 Nor Flash 具有显著的性能优势：读取速度可达 200-400 MB/s，访问延迟低至 70-120ns，可以直接映射到处理器地址空间实现 XIP（Execute-In-Place）代码执行。

然而，并行接口也面临着一些挑战。首先是引脚数量众多，一个典型的 x16 并行 Nor Flash 需要 30-56 个引脚，包括地址总线、数据总线、控制信号等，这增加了 PCB 布局的复杂性和成本。其次是信号完整性问题，多根并行信号线需要严格的时序对齐，布线长度差异过大会影响信号质量。此外，并行接口的功耗通常高于串行接口，在移动设备等对功耗敏感的场景中需要权衡。

**串行接口特点**：

串行接口（如 SPI）使用单根或少数几根数据线顺序传输数据。SPI 接口只需 4-6 根信号线（时钟、片选、数据输入输出），大大简化了硬件设计，降低了封装成本和 PCB 布线难度。随着技术的演进，SPI 接口的性能也在不断提升：Quad SPI 可达到 66.6 MB/s 的读取速度，Octal SPI DDR 模式更可达 400 MB/s，基本接近并行接口的性能水平。

串行接口的优势还包括：更少的引脚意味着更小的封装尺寸（SOP8、USON8 等），更适合空间受限的应用；简化的布线降低了信号完整性设计的难度；功耗通常低于并行接口。但串行接口的访问延迟较高，不适合对延迟极其敏感的应用场景。

**接口选择建议**：

选择并行还是串行接口需要综合考虑多种因素。对于需要高性能代码执行、XIP 启动或高速数据读取的应用，并行接口是更好的选择。对于成本敏感、空间受限或对功耗有严格要求的应用，SPI 接口更为合适。现代系统设计中，经常可以看到两种接口并存的情况：SPI Flash 用于低成本的启动代码存储，并行 Flash 用于大容量的程序代码或数据存储。

### 并行 Nor Flash 的应用场景

并行 Nor Flash 凭借其高带宽、低延迟的特性，在多个关键应用领域发挥着重要作用。

**FPGA 配置存储**：

这是并行 Nor Flash 最重要的应用领域之一。FPGA 器件在上电后需要从外部存储器加载配置文件来配置内部的逻辑单元。并行 Nor Flash 提供了高速的配置数据读取能力，Xilinx 7 系列 FPGA 支持从并行 Nor Flash 进行配置，配置速度可达数百 Mbps。FPGA 配置通常采用被动并行（Parallel Passive）或被动快速（Fast Passive）模式，需要 Flash 芯片支持特定的时序要求。

**嵌入式系统代码存储**：

在高性能嵌入式系统中，并行 Nor Flash 常用于存储需要快速访问的程序代码。处理器可以直接从并行 Nor Flash 中执行代码（XIP），无需先将代码复制到 RAM 中运行，这对于启动加载器和实时性要求高的系统尤为重要。汽车电子、工业控制等领域对系统启动速度和可靠性要求较高，常采用并行 Nor Flash 作为启动代码存储介质。

**高性能数据存储**：

某些应用需要频繁读取大量数据，如多媒体处理、图像缓冲等。并行 Nor Flash 的高带宽特性使其可以作为高速数据存储介质使用。虽然写入速度不如 Nand Flash，但对于写入操作较少、以读取为主的应用场景，并行 Nor Flash 是理想选择。

**网络设备和通信系统**：

网络设备（如路由器、交换机）需要在启动时快速加载大量的配置代码和固件，并行 Nor Flash 能够满足这种高速启动需求。通信系统中的一些专用芯片（如 DSP、基带处理器）也常常使用并行接口访问外部 Nor Flash 存储代码和配置数据。

### 主流并行 Nor Flash 芯片

市场上主要的 Nor Flash 厂商都提供了丰富的并行接口产品线，开发者可以根据容量、速度、电压等参数进行选择。

**Winbond（华邦电子）**：

Winbond 的 W29GL 系列是经典的并行 Nor Flash 产品线，支持 CFI 标准接口，提供 x8/x16 可选数据宽度。代表产品包括 W29GL064CHb（约 64Mbit）、W29GL128CHb（约 128Mbit）、W29GL256CHb（约 256Mbit）等。该系列支持 3.3V 单电源供电，读取速度可达 90-120ns。

**Macronix（旺宏电子）**：

Macronix 的 MX29GL 系列同样提供丰富的容量选择，支持 CFI 接口标准，具有良好的兼容性。MX29GL 系列采用先进的制程技术，提供高速读取能力和低功耗特性。典型型号包括 MX29GL640G（64Mbit）、MX29GL128G（128Mbit）、MX29GL512G（512Mbit）等。

**Micron（镁光科技）**：

Micron 的 MT28EW 和 MT28AP 系列是面向高性能应用的并行 Nor Flash，采用先进的制造工艺，提供卓越的读写性能和可靠性。MT28EW 系列支持 1.8V 和 3.3V 电压选项，容量从 256Mbit 到 1Gbit 不等，是大容量并行存储的理想选择。

**ISSI（芯成半导体）**：

ISSI 的 IS66WVH 和 IS67WVH 系列提供高性价比的并行 Nor Flash 解决方案，支持 CFI 标准，具有稳定的供货能力和完整的车规级产品认证。该系列适合对成本敏感但需要可靠存储的应用。

## 并行接口信号定义

并行 Nor Flash 使用一组标准化的信号进行数据传输和控制。理解这些信号的功能和时序关系是正确使用并行接口的基础。以下是典型并行 Nor Flash 的信号定义。

### 信号总览

以典型的 56 引脚 TSOP 封装并行 Nor Flash 为例，其信号引脚可分为以下几类：地址总线、数据总线、控制信号和电源/地引脚。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    并行 Nor Flash 信号引脚分布                      │
├─────────────────────────────────────────────────────────────────────┤
│  地址总线:  A0-A25 (26 根)                                          │
│  数据总线:  DQ0-DQ15 (16 根)                                        │
│  控制信号:  CE#, OE#, WE#, RESET#, RY/BY#, WP#, BYTE#              │
│  电源/地:   VCC, VSS (多组)                                          │
└─────────────────────────────────────────────────────────────────────┘
```

### 地址总线

地址总线用于指定要访问的 Flash 内部存储单元地址。地址总线的宽度取决于 Flash 芯片的容量和寻址模式。

**地址引脚定义**：

| 信号名称 | 方向 | 功能描述 |
|---------|------|----------|
| A0-A25 | 输入 | 地址总线，共 26 位，支持最大 64MB 寻址空间（x8 模式） |

在 x16 模式下，A0 地址引脚通常被用作数据校准控制（BYTE# 信号），实际地址总线为 A1-A25，共 25 位。这种设计使得处理器可以通过调整地址来访问 x16 模式下的高低字节。

**地址信号特性**：

地址总线采用标准的 CMOS 输入电平，逻辑高电平通常要求高于 0.7×VCC，逻辑低电平要求低于 0.3×VCC。地址信号的建立时间（tAS）和保持时间（tAH）是时序参数中的重要指标，在高速访问时需要严格满足这些时序要求。

地址总线的负载能力需要根据连接的主控设备数量和布线长度进行评估。对于长距离传输或连接多个负载的情况，可能需要添加地址缓冲器来确保信号完整性。

### 数据总线

数据总线用于在 Flash 芯片和主控之间传输实际的数据内容。数据总线的宽度决定了每次传输的数据量。

**数据引脚定义**：

| 信号名称 | 方向 | 功能描述 |
|---------|------|----------|
| DQ0-DQ7 | 双向 | 低 8 位数据总线（x8/x16 模式共用） |
| DQ8-DQ15 | 双向 | 高 8 位数据总线（仅 x16 模式使用） |

在 x8 模式下，只使用 DQ0-DQ7 进行数据传输，DQ8-DQ15 可以悬空或被内部禁用。这种灵活性使得同一颗 Flash 芯片可以在不同的系统中使用不同的数据宽度。

**数据总线特性**：

数据总线是双向的，在读取操作时 Flash 芯片输出数据，在写入操作时接收主控发送的数据。数据总线在芯片未被选中（CE# 无效）或输出被禁用（OE# 无效）时处于高阻态，避免总线冲突。

数据总线的信号完整性设计至关重要，尤其是在高速传输时。合理的长度匹配、阻抗控制和端接策略可以有效减少信号反射和串扰。建议在 PCB 设计时将数据线作为等长走线处理，控制组内信号的长度差异在一定范围内。

### 控制信号

并行 Nor Flash 使用多个控制信号来协调地址、数据的传输以及读/写操作的执行。

**核心控制信号定义**：

| 信号名称 | 方向 | 极性 | 功能描述 |
|---------|------|------|----------|
| CE# | 输入 | 低有效 | 片选信号（Chip Enable），选中该芯片使其参与总线操作 |
| OE# | 输入 | 低有效 | 输出使能信号（Output Enable），控制数据总线输出 |
| WE# | 输入 | 低有效 | 写使能信号（Write Enable），指示写入操作 |
| RESET# | 输入 | 低有效 | 硬件复位信号，用于将芯片复位到初始状态 |

**CE# 片选信号**：

片选信号 CE# 用于激活 Flash 芯片，使其参与总线操作。当 CE# 为低电平时，Flash 芯片被选中并响应地址总线上的访问请求；当 CE# 为高电平时，芯片进入低功耗待机状态，数据总线呈高阻态。

在多 Flash 系统中，可以通过对 CE# 的控制来实现芯片选择。片选信号的时序要求包括：地址有效后 CE# 必须建立，CE# 必须在数据有效前建立并保持一段时间。具体时序参数因芯片型号而异，需要参考器件数据手册。

**OE# 输出使能信号**：

输出使能信号 OE# 控制 Flash 芯片将数据放到数据总线上。当 OE# 为低电平时，如果芯片已被 CE# 选中且地址有效，芯片将输出相应的数据；当 OE# 为高电平时，数据总线呈高阻态。

OE# 信号通常由处理器的读信号直接控制或通过译码电路产生。在某些系统中，OE# 可以始终保持低电平，由 CE# 单独控制数据的输出使能。

**WE# 写使能信号**：

写使能信号 WE# 指示当前的访问是写入操作。当 WE# 为低电平时，地址总线上的值被锁存为目标地址，数据总线上的值被锁存为要写入的数据。

写操作的时序比读操作更为复杂，需要满足地址建立时间、数据建立时间和数据保持时间等多项时序参数。写入操作通常需要数微秒到数十微秒的时间完成，期间需要通过 RY/BY# 信号或状态寄存器轮询来确认操作完成。

**辅助控制信号定义**：

| 信号名称 | 方向 | 极性 | 功能描述 |
|---------|------|------|----------|
| RY/BY# | 输出 | 低有效 | 就绪/忙碌指示（Ready/Busy），低电平表示芯片正在执行内部操作 |
| WP# | 输入 | 低有效 | 写保护信号（Write Protect），低电平时锁定部分或全部存储区 |
| BYTE# | 输入 | - | 字节/字模式选择，高电平为 x16 模式，低电平为 x8 模式 |

**RY/BY# 就绪/忙碌信号**：

RY/BY# 是一个漏极开路输出信号，需要外部上拉电阻。在 Flash 芯片执行擦除、编程等内部操作时，RY/BY# 输出低电平表示芯片忙碌；操作完成后，RY/BY# 变为高电平表示就绪。

该信号可以用于轮询等待操作完成，也可以配置为中断源以实现异步通知。使用时需要注意上拉电阻的阻值选择，过大的阻值可能导致噪声干扰问题。

**WP# 写保护信号**：

WP# 信号用于硬件写保护功能。当 WP# 为低电平时，芯片的某些区域可能被锁定，防止意外写入。具体的保护机制因芯片型号而异，部分芯片支持通过 WP# 实现整个芯片的写保护。

**BYTE# 字节模式选择**：

BYTE# 信号用于选择 Flash 芯片的工作模式。当 BYTE# 为高电平时，芯片工作于 x16 模式，数据总线使用全部 16 位（DQ0-DQ15）；当 BYTE# 为低电平时，芯片工作于 x8 模式，只使用低 8 位（DQ0-DQ7），DQ8-DQ15 用于 A-1 地址输入。

## 异步并行接口时序

并行 Nor Flash 采用异步接口方式，没有独立的时钟信号，数据的传输完全由控制信号的时序关系决定。理解异步时序是正确配置 Flash 控制器和调试接口问题的关键。

### 读时序周期

读取操作是 Nor Flash 最常用的操作，优化的读时序可以实现尽可能高的读取带宽。

**典型异步读时序图**：

```
                    读操作时序图
    ┌─────────────────────────────────────────────────────────────┐
    │                                                              │
    │  CE#  ───┬─────────────────────────────────────┐            │
    │          │                                     │            │
    │       ───┘                                     └────        │
    │                                                              │
    │  OE#   ───┬─────────────────────────────────────┐            │
    │          │                                     │            │
    │       ───┘                                     └────        │
    │                                                              │
    │  A    ──────────[Valid Address]────────────────→           │
    │          │     │                              │            │
    │          │ tAVQV                              │            │
    │          └─────┴──────────────────────────────┘            │
    │                                                              │
    │  DQ    ───────────────────[Valid Data]────────────→        │
    │                        │        │              │            │
    │                        │ tELQV  | tOH          │            │
    │                        └────────┴──────────────┘            │
    │                                                              │
    │  说明:                                                       │
    │  tAVQV: 地址有效到数据有效（访问时间）                        │
    │  tELQV: CE# 低电平到数据有效                                  │
    │  tOH:   输出保持时间                                         │
    │  tOHZ:  OE# 高电平后数据高阻时间                             │
    │  tELOX: CE# 低电平到 OE# 低电平的建立时间                    │
    │                                                              │
    └─────────────────────────────────────────────────────────────┘
```

**读时序步骤详解**：

1. **地址建立阶段**：主控将目标地址驱动到地址总线（A0-A25）上。地址信号需要稳定一段时间（地址建立时间 tAS）后才能有效读取。

2. **片选激活**：主控将 CE# 信号置低，选中 Flash 芯片。片选信号的有效需要在地址有效之后或同时发生。

3. **输出使能**：主控将 OE# 信号置低，允许 Flash 芯片输出数据。OE# 必须在 CE# 有效后一段时间内（tELOX）建立。

4. **数据访问**：Flash 芯片在接收到有效的地址和读控制信号后，开始内部数据访问。从地址有效到数据输出的时间称为访问时间（tAVQV 或 tACC），这是决定读取速度的关键参数。

5. **数据读取**：数据在总线上变得有效后，主控可以在此时采样数据。数据需要保持一段时间（tOH）以确保主控正确采样。

6. **操作结束**：主控将 OE# 和 CE# 置高，Flash 芯片的数据输出进入高阻态。

**关键时序参数**：

| 参数名称 | 符号 | 典型值 | 描述 |
|---------|------|--------|------|
| 访问时间 | tACC | 70-120ns | 从地址有效到数据有效的时间 |
| 片选使能到数据有效 | tELQV | 70-120ns | 从 CE# 低电平到数据有效的时间 |
| 输出使能到数据有效 | tGLQV | 10-30ns | 从 OE# 低电平到数据有效的时间 |
| 输出保持时间 | tOH | 0-10ns | 数据在 OE# 或 CE# 变高后的保持时间 |
| 输出禁用延迟 | tOHZ | 10-30ns | OE# 变高后数据变为高阻态的延迟 |
| 地址建立时间 | tAS | 5-10ns | 地址有效到 CE# 或 WE# 变低的建立时间 |
| 地址保持时间 | tAH | 5-10ns | CE# 或 WE# 变高后地址的保持时间 |

### 写时序周期

写入操作比读取操作更为复杂，需要严格按照时序要求进行，否则可能导致写入失败或数据错误。

**典型异步写时序图**：

```
                    写操作时序图
    ┌─────────────────────────────────────────────────────────────┐
    │                                                              │
    │  CE#  ───┬─────────────────────────────────────┐            │
    │          │                                     │            │
    │       ───┘                                     └────        │
    │                                                              │
    │  WE#   ───┬─────────────────┐  ┌─────────────────┐          │
    │          │                 │  │                 │          │
    │       ───┘                 └──┘                 └────       │
    │              │              │      │                          │
    │              │  tDS          │ tDH  │                          │
    │  A    ────────[Addr]───────────────[Next Addr]────────→     │
    │          │              │      │      │                          │
    │  DQ    ───────────────────────────────────────────────→     │
    │          │          [Data]         │                          │
    │          │          │  tWP         │                          │
    │          └──────────┴──────────────┴───────────────          │
    │                                                              │
    │  说明:                                                       │
    │  tAS:  地址建立时间                                          │
    │  tAH:  地址保持时间                                          │
    │  tDS:  数据建立时间（数据相对于 WE# 下降沿）                  │
    │  tDH:  数据保持时间（数据相对于 WE# 上升沿）                  │
    │  tWP:  写脉冲宽度                                             │
    │  tCS:  片选建立时间                                           │
    │  tCH:  片选保持时间                                           │
    │                                                              │
    └─────────────────────────────────────────────────────────────┘
```

**写时序步骤详解**：

1. **地址建立**：主控将目标地址驱动到地址总线上，地址需要满足建立时间 tAS 的要求。

2. **数据建立**：主控将待写入的数据驱动到数据总线上。数据必须在 WE# 下降沿之前达到稳定状态，满足数据建立时间 tDS 的要求。

3. **写脉冲**：主控将 WE# 信号置低，启动写入操作。WE# 的脉冲宽度必须满足 tWP 的要求，不能过窄。

4. **数据保持**：在 WE# 上升沿之后，数据需要继续保持稳定一段时间（tDH），确保数据被正确锁存。

5. **内部编程**：WE# 上升沿触发 Flash 芯片内部的编程操作。此时地址和数据被锁存，芯片开始执行写入。

6. **等待完成**：芯片执行内部写入操作期间，RY/BY# 信号保持低电平。主控需要轮询 RY/BY# 或状态寄存器来等待操作完成。

7. **操作完成**：写入操作完成后，RY/BY# 变高，主控可以开始下一次访问。

**关键时序参数**：

| 参数名称 | 符号 | 典型值 | 描述 |
|---------|------|--------|------|
| 写脉冲宽度 | tWP | 40-100ns | WE# 脉冲宽度 |
| 数据建立时间 | tDS | 20-40ns | 数据相对于 WE# 下降沿的建立时间 |
| 数据保持时间 | tDH | 5-10ns | 数据相对于 WE# 上升沿的保持时间 |
| 地址建立时间 | tAS | 0-10ns | 地址相对于 WE# 下降沿的建立时间 |
| 地址保持时间 | tAH | 40-60ns | 地址相对于 WE# 上升沿的保持时间 |
| 片选建立时间 | tCS | 0ns | CE# 相对于 WE# 下降沿的建立时间 |
| 片选保持时间 | tCH | 0ns | CE# 相对于 WE# 上升沿的保持时间 |
| 编程时间 | tWP | 10-200us | 字节/字编程时间（取决于具体芯片） |

### 时序参数设置示例

在实际系统中，时序参数的设置需要根据主控的时钟频率和 Flash 芯片的时序要求进行计算和调整。

**STMicroelectronics STM32 FSMC 时序配置**：

STM32 的 FSMC（Flexible Static Memory Controller）提供了可编程的时序参数，适合与并行 Nor Flash 接口。以下是典型配置示例：

```
FSMC_NORSRAM_TimingTypeDef Timing;

/* 读取时序配置 */
Timing->AddressSetupTime = 1;      /* 地址建立时间: 1个HCLK */
Timing->AddressHoldTime = 1;       /* 地址保持时间: 1个HCLK */
Timing->DataSetupTime = 3;         /* 数据建立时间: 3个HCLK */
Timing->BusTurnAroundDuration = 1; /* 总线转向时间: 1个HCLK */
Timing->CLKDivision = 0;           /* 时钟分频（用于同步模式） */
Timing->DataLatency = 0;           /* 数据延迟（用于同步模式） */
Timing->AccessMode = FSMC_ACCESS_MODE_A; /* 异步模式A */

/* 写入时序配置 */
Timing->AddressSetupTime = 1;
Timing->AddressHoldTime = 1;
Timing->DataSetupTime = 2;         /* 写入需要更短的数据建立时间 */
Timing->BusTurnAroundDuration = 1;
Timing->CLKDivision = 0;
Timing->DataLatency = 0;
Timing->AccessMode = FSMC_ACCESS_MODE_A;
```

假设 HCLK 为 100MHz（10ns），则：
- 地址建立时间 = 1 × 10ns = 10ns
- 数据建立时间（读）= 3 × 10ns = 30ns
- 数据建立时间（写）= 2 × 10ns = 20ns

这些值需要根据实际 Flash 芯片的时序参数进行调整。如果 HCLK 较高，可能需要增加时序参数的值以满足 Flash 的时序要求。

## x8/x16 数据宽度

并行 Nor Flash 支持 x8（8位）和 x16（16位）两种数据宽度模式，不同的数据宽度会影响地址映射、访问粒度和系统性能。

### x8 模式（8位数据总线）

在 x8 模式下，Flash 芯片使用 8 位数据总线（DQ0-DQ7）进行数据传输。这种模式是最基本的操作模式，几乎所有的并行 Nor Flash 都支持 x8 模式。

**x8 模式特点**：

- 数据总线只使用 DQ0-DQ7，共 8 根信号线
- 每个地址对应一个字节（8 位）的存储空间
- 地址总线全部可用（A0-A25），最大寻址空间为 64MB
- 引脚数量较少，PCB 布线相对简单
- 兼容性最好，几乎所有控制器都支持 x8 模式

**x8 模式地址映射**：

在 x8 模式下，地址与存储单元一一对应。例如，地址 0x0000 访问第一个字节，地址 0x0001 访问第二个字节，以此类推。这种直接映射方式简单直观，软件开发方便。

```
x8 模式地址映射示例：
┌──────────────────┬──────────────────┐
│   Flash 地址     │   对应存储单元    │
├──────────────────┼──────────────────┤
│   0x000000       │   第 1 字节       │
│   0x000001       │   第 2 字节       │
│   0x000002       │   第 3 字节       │
│   ...            │   ...             │
│   0x00FFFF       │   第 65536 字节   │
└──────────────────┴──────────────────┘
```

### x16 模式（16位数据总线）

在 x16 模式下，Flash 芯片使用全部 16 位数据总线（DQ0-DQ15）进行数据传输，可以在一个访问周期内读取或写入 16 位（2 字节）的数据。

**x16 模式特点**：

- 数据总线使用全部 16 根信号线（DQ0-DQ15）
- 每个地址对应一个 16 位字（2 字节）的存储空间
- A0 地址引脚被用作 BYTE# 模式选择，A1-A25 作为地址输入
- 最大寻址空间为 32MB（64M 字）
- 理论带宽是 x8 模式的两倍
- 需要主控支持 16 位数据总线

**x16 模式地址映射**：

在 x16 模式下，地址按字（16 位）为单位递增。每个地址对应一个 16 位字，而不是字节。这意味着地址 0x0000 访问前两个字节（低字节 DQ0-DQ7 和高字节 DQ8-DQ15），地址 0x0001 访问接下来的两个字节。

```
x16 模式地址映射示例：
┌──────────────────┬─────────────────────┬─────────────────────┐
│   Flash 地址     │   低字节 (DQ0 │   高字节-7)    (DQ8-15)   │
├──────────────────┼─────────────────────┼─────────────────────┤
│   0x000000       │   第 1 字节         │   第 2 字节         │
│   0x000001       │   第 3 字节         │   第 4 字节         │
│   0x000002       │   第 5 字节         │
│   ...            │   ...               │   ...               │
└──────────────────┴─────────────────────┴─────────────────────┘
```

**地址对齐要求**：

在 x16 模式下，处理器进行 16 位访问时通常要求地址对齐。即 16 位访问的起始地址必须是偶数地址（字对齐）。如果进行字节访问，需要使用特定的字节使能信号（UB/LB）来控制高字节或低字节的写入。

### 模式切换方法

Flash 芯片的数据宽度模式由 BYTE# 引脚的电平状态决定，在硬件设计时需要根据系统需求选择合适的模式。

**BYTE# 引脚功能**：

| BYTE# 状态 | 工作模式 | 说明 |
|-----------|----------|------|
| 高电平 (VCC) | x16 模式 | 16 位数据总线，所有地址线可用 |
| 低电平 (GND) | x8 模式 | 8 位数据总线，A0 作为最低位地址 |

**模式选择注意事项**：

1. **硬件连接**：BYTE# 引脚在硬件设计时就需要确定，通常直接连接到 VCC 或 GND，不需要动态切换。如果需要同时支持两种模式，可能需要使用跳线或可配置电阻。

2. **地址计算**：在软件层面，需要根据模式正确计算地址。x8 模式下地址直接对应字节；x16 模式下，处理器发送的字节地址需要除以 2（或右移 1 位）才能得到 Flash 的字地址。

3. **数据组织**：x16 模式下的数据组织为小端格式（Little Endian）还是大端格式（Big Endian）取决于具体芯片和数据总线连接方式，需要在软件中正确处理。

4. **混合访问**：某些系统可能需要在 x16 模式下进行字节级访问。这通常通过控制特定的高字节使能（UB#）和低字节使能（LB#）信号来实现，但这需要 Flash 芯片支持相应的功能。

## FPGA 专用接口

并行 Nor Flash 在 FPGA 配置领域有广泛应用。Xilinx 7 系列 FPGA 支持多种配置模式，包括被动串行（PS）、被动并行（PP）和被动快速（Fast PASS）模式。本节详细介绍 FPGA 配置接口的时序要求和配置方法。

### Xilinx 7 系列 FPGA 配置接口

Xilinx 7 系列 FPGA（Artix-7、Kintex-7、Virtex-7）提供了灵活的配置接口，支持从多种外部存储器加载配置数据。并行 Nor Flash 是最常用的配置存储器之一。

**配置接口信号定义**：

| 信号名称 | 方向 | 描述 |
|---------|------|------|
| CCLK | 输入/输出 | 配置时钟，由 FPGA 输出（Master 模式）或外部提供（Slave 模式） |
| D[0:31] | 双向 | 配置数据总线，PP 模式使用 D[0:7] |
| FCS# | 输出 | Flash 片选信号，低有效 |
| FOE# | 输出 | Flash 输出使能信号，低有效 |
| FWE# | 输出 | Flash 写使能信号，低有效 |
| RDWR# | 输入 | 读/写模式选择 |
| INIT# | 双向 | 初始化完成指示，配置前为输入 |
| DONE | 输出 | 配置完成指示 |

**配置模式分类**：

Xilinx 7 系列 FPGA 支持以下主要配置模式：

1. **主动串行配置（Master SPI）**：FPGA 作为主设备，从串行 SPI Flash 读取配置数据。这是最常用的配置方式，使用 1 根数据线（DI/DO），速度相对较慢但引脚最少。

2. **主动并行配置（Master Parallel）**：FPGA 作为主设备，从并行 Nor Flash 读取配置数据。使用 8 位或 16 位数据总线，速度比 SPI 快。

3. **被动串行配置（Slave Serial）**：外部处理器作为主设备，通过串行方式向 FPGA 发送配置数据。

4. **被动并行配置（Slave Parallel）**：外部处理器作为主设备，通过并行方式向 FPGA 发送配置数据。速度最快，但引脚数量最多。

5. **被动快速配置（Fast Passive Parallel）**：高速被动并行配置模式，数据传输速率更高。

### 配置时序要求

FPGA 从并行 Nor Flash 读取配置数据时的时序要求比普通读取操作更为严格，需要确保数据在 FPGA 指定的时序窗口内稳定。

**被动并行配置时序**：

```

    │       ───┘       └───────┘       └───────┘       └────     │
    │                                                              │

    │         │                                   │              │

    │                                                              │
```

| tOH | 1.5ns | - | CCLK 上升沿后数据保持时间 |

### Fast PASS 模式

Fast PASS（被动快速并行）模式是 Xilinx 7 系列 FPGA 的高速配置模式，提供比标准被动并行模式更高的配置速度。

**Fast PASS 模式特点**：

- 支持高达 200MHz 的配置时钟频率
- 使用 8 位或 16 位数据总线
- 数据在 CCLK 的下降沿输出
- 需要 Flash 芯片支持快速读取操作

**配置时钟频率与时序**：

| 配置模式 | 最大时钟频率 | 典型时序需求 |

选择支持相应时序要求的 Flash 芯片对于实现高速配置至关重要。Micron、Macronix 等厂商的某些并行 Nor Flash 型号专门针对 FPGA 配置进行了优化，提供更快的读取速度。

## FSMC/FMC 配置（针对 MCU）

STM32 系列微控制器的 FSMC（Flexible Static Memory Controller）和 FMC（Flexible Memory Controller）提供了与外部存储器和并行 Nor Flash 接口的硬件支持。本节详细介绍 FSMC/FMC 的配置方法和时序参数设置。

### FSMC 简介

FSMC 是 STMicroelectronics STM32 系列 MCU 的外部存储器控制器，支持与多种类型存储器的接口，包括 SRAM、NOR Flash、NAND Flash 和 PC Card。FSMC 通过可编程的时序参数，可以适配不同速度要求的外部存储器。

**FSMC 特性**：

- 支持 8 位和 16 位数据总线宽度
- 多达 4 个存储区域（Bank），每个区域可独立配置
- 可编程的时序参数（地址建立/保持、数据建立等）
- 支持异步访问和同步访问模式
- 支持 NAND Flash 的 ECC 功能

**FSMC 存储区域划分**：

FSMC 将外部存储器映射到处理器的地址空间，四个存储区域（Bank1-Bank4）分别对应不同的地址范围。以 STM32F4 为例：

| 存储区域 | 地址范围 | 典型应用 |
|---------|----------|----------|
| Bank1 | 0x6000 0000 - 0x6FFF FFFF | NOR Flash / SRAM |
| Bank2 | 0x7000 0000 - 0x7FFF FFFF | NAND Flash 1 |
| Bank3 | 0x8000 0000 - 0x8FFF FFFF | NAND Flash 2 |
| Bank4 | 0x9000 0000 - 0x9FFF FFFF | PC Card |

Bank1 专门用于 NOR Flash 和 SRAM，可进一步划分为 4 个子区域（1MB 每个）。

### FMC 简介

FMC 是 STM32F7 和 STM32H7 系列 MCU 的增强型存储器控制器，在 FSMC 的基础上增加了更多功能和改进。FMC 与 FSMC 在基本概念上相似，但提供了更高的性能和更多配置选项。

**FMC 主要特性****：

- 更高的时钟频率支持（高达 100MHz）
- 独立的时钟使能信号
- 增强的时序控制
- 支持同步 NOR Flash 和移动 NAND
- 支持 eMMC/SD 卡接口

**FSMC 与 FMC 兼容性**：

FSMC 的配置代码在大多数情况下可以迁移到 FMC，但需要根据具体型号进行调整。FMC 的寄存器结构和时序参数设置比 FSMC 更复杂，但也提供了更大的灵活性。

### 寄存器配置

以下是 STM32 FSMC 配置并行 Nor Flash 的详细步骤和寄存器设置。

**FSMC 关键寄存器**：

| 寄存器 | 功能 |
|--------|------|
| FSMC_Bank1->BTCR[0-1] | Bank1 控制寄存器（模式、时序） |
| FSMC_BTR[0-3] | Bank1 时序寄存器（读） |
| FSMC_BWTR[0-3] | Bank1 时序寄存器（写） |
| FSMC_PCRx | NAND/PC Card 控制寄存器 |

**Bank1 NOR/PSRAM 控制寄存器（BCR）配置**：

```
FSMC_NORSRAM_InitTypeDef FSMC_NORSRAMInit;

/* 基础配置 */
FSMC_NORSRAMInit->NSBank = FSMC_NORSRAM_BANK1;      /* Bank1 */
FSMC_NORSRAMInit->DataAddressMux = FSMC_DATA_ADDRESS_MUX_DISABLE; /* 非复用模式 */
FSMC_NORSRAMInit->MemoryType = FSMC_MEMORY_TYPE_NOR; /* NOR Flash */
FSMC_NORSRAMInit->MemoryDataWidth = FSMC_NORSRAM_MEM_BUS_WIDTH_16; /* x16 */
FSMC_NORSRAMInit->BurstAccessMode = FSMC_BURST_ACCESS_MODE_DISABLE; /* 异步模式 */
FSMC_NORSRAMInit->WaitSignalPolarity = FSMC_WAIT_SIGNAL_POLARITY_LOW;
FSMC_NORSRAMInit->WaitSignalActive = FSMC_WAIT_TIMING_BEFORE_WS;
FSMC_NORSRAMInit->WriteOperation = FSMC_WRITE_OPERATION_ENABLE;
FSMC_NORSRAMInit->WaitSignal = FSMC_WAIT_SIGNAL_DISABLE;
FSMC_NORSRAMInit->ExtendedMode = FSMC_EXTENDED_MODE_DISABLE;
FSMC_NORSRAMInit->AsynchronousWait = FSMC_ASYNCHRONOUS_WAIT_DISABLE;
```

**Bank1 NOR/PSRAM 时序寄存器（BTR）配置**：

```
/* 读取时序 */
FSMC_NORSRAMInit->ReadWriteTimingStruct->AddressSetupTime = 1;      /* 1个HCLK周期 */
FSMC_NORSRAMInit->ReadWriteTimingStruct->AddressHoldTime = 1;        /* 1个HCLK周期 */
FSMC_NORSRAMInit->ReadWriteTimingStruct->DataSetupTime = 3;          /* 3个HCLK周期 */
FSMC_NORSRAMInit->ReadWriteTimingStruct->BusTurnAroundDuration = 1;  /* 1个HCLK周期 */
FSMC_NORSRAMInit->ReadWriteTimingStruct->CLKDivision = 0;           /* 同步模式使用 */
FSMC_NORSRAMInit->ReadWriteTimingStruct->DataLatency = 0;           /* 同步模式使用 */
FSMC_NORSRAMInit->ReadWriteTimingStruct->AccessMode = FSMC_ACCESS_MODE_A;
```

**写入时序配置**：

如果要使用不同的读取和写入时序，需要启用扩展模式：

```
FSMC_NORSRAMInit->ExtendedMode = FSMC_EXTENDED_MODE_ENABLE;

/* 写入时序单独配置 */
FSMC_NORSRAMInit->WriteTimingStruct->AddressSetupTime = 1;
FSMC_NORSRAMInit->WriteTimingStruct->AddressHoldTime = 1;
FSMC_NORSRAMInit->WriteTimingStruct->DataSetupTime = 2;  /* 写入通常需要更短的时间 */
FSMC_NORSRAMInit->WriteTimingStruct->BusTurnAroundDuration = 1;
FSMC_NORSRAMInit->WriteTimingStruct->AccessMode = FSMC_ACCESS_MODE_A;
```

### 时序参数计算

FSMC 的时序参数需要根据 HCLK 时钟频率和 Flash 芯片的时序要求进行计算。

**计算示例**：

假设：
- HCLK = 100MHz（周期 = 10ns）
- Flash 芯片参数：
  - tACC（访问时间）= 90ns
  - tDS（数据建立时间）= 25ns
  - tDH（数据保持时间）= 5ns
  - tAS（地址建立时间）= 5ns

**参数计算**：

1. **DataSetupTime（数据建立时间）**：
   - 需要满足 Flash 的 tDS 要求
   - DataSetupTime × HCLK >= tDS
   - DataSetupTime >= 25ns / 10ns = 2.5，取整数 3
   - 实际时间 = 3 × 10ns = 30ns（满足要求）

2. **AddressSetupTime（地址建立时间）**：
   - 需要满足 Flash 的 tAS 要求
   - AddressSetupTime × HCLK >= tAS
   - AddressSetupTime >= 5ns / 10ns = 0.5，取整数 1
   - 实际时间 = 1 × 10ns = 10ns（满足要求）

3. **总读取周期**：
   - 总时间 = (AddressSetupTime + AddressHoldTime + DataSetupTime) × HCLK
   - = (1 + 1 + 3) × 10ns = 50ns
   - 这对应 20MHz 的有效访问频率

**时序调整建议**：

- 如果 HCLK 较高，可能需要增加 AddressSetupTime 和 DataSetupTime
- 写入操作通常比读取快，可以适当减小 WriteDataSetupTime
- 如果出现数据错误，首先尝试增加 DataSetupTime
- 布局布线引起的信号延迟也需要考虑，必要时增加时序余量

## 地址/数据总线复用

并行 Nor Flash 支持地址总线和数据总线复用（Multiplexed）和非复用（Non-Multiplexed）两种模式。不同的复用模式会影响系统设计复杂性和引脚数量。

### 复用模式

在复用模式下，地址总线和数据总线共享相同的物理引脚，通过分时复用的方式传输地址和数据。这种设计可以显著减少引脚数量，尤其适合引脚资源受限的系统。

**复用模式特点**：

- 地址线与数据线共用引脚，减少引脚数量约 50%
- 需要外部地址锁存器（如 74HC573）来保存地址
- 增加了电路复杂度
- 访问速度略低于非复用模式（由于地址锁存开销）

### 非复用模式

在非复用模式下，地址总线和数据总线使用独立的物理引脚。这种模式是最常用的连接方式，提供最简单的硬件设计和最高的访问性能。

**非复用模式特点**：

- 地址和数据使用独立引脚，无需锁存器
- 硬件设计简单，调试方便
- 性能最优，无需额外的地址锁存周期
- 引脚数量较多

**非复用模式接口连接**：

### 优缺点分析

选择复用模式还是非复用模式需要根据具体应用场景进行权衡。

**非复用模式优势**：

| 优势 | 说明 |
|------|------|
| 设计简单 | 无需外部锁存器，硬件设计更简洁 |
| 性能最高 | 无需额外的地址锁存周期，访问速度最快 |
| 可靠性高 | 信号数量少，信号完整性问题少 |
| 易于调试 | 时序直观，调试相对容易 |

**非复用模式劣势**：

| 劣势 | 说明 |
|------|------|
| 引脚多 | 需要大量 GPIO，对于引脚资源有限的 MCU 可能是问题 |
| PCB 面积大 | 并行信号走线需要更多 PCB 空间 |
| 成本高 | 封装更大的 Flash 芯片成本更高 |

**复用模式优势**：

| 优势 | 说明 |
|------|------|
| 引脚少 | 地址和数据共用，显著减少引脚数量 |
| PCB 紧凑 | 走线数量减少，PCB 布局更紧凑 |
| 成本低 | 可能使用更小封装的 Flash 芯片 |

**复用模式劣势**：

| 劣势 | 说明 |
|------|------|
| 设计复杂 | 需要地址锁存器和额外的控制信号 |
| 性能降低 | 需要额外的地址锁存周期 |
| 可靠性问题 | 更多的信号意味着更多的潜在问题点 |
| 调试困难 | 时序更复杂，问题诊断更困难 |

**选择建议**：

- 对于大多数应用，推荐使用非复用模式，尤其是 STM32 等带有 FSMC 控制器的 MCU
- 当 MCU 的 GPIO 引脚数量有限时，可以考虑复用模式
- 对于高性能应用（如 FPGA 配置），应使用非复用模式以获得最佳性能
- 如果系统需要支持多种类型的外部存储器，非复用模式更具灵活性

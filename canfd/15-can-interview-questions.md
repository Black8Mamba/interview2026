# 常见面试题汇总

本章汇总 CAN/CANFD 协议的高频面试题，按知识点分类，便于面试前快速复习。

---

## 8.1 基础概念类

### Q1: CAN 总线采用什么方式进行通信？有什么特点？

**参考答案**：
- **通信方式**：多主通信、串行通信、差分信号传输
- **特点**：
  - 多主通信：所有节点都可以发起通信
  - 非破坏性仲裁：高优先级消息优先发送
  - 差分信号：抗干扰能力强
  - 错误检测：多种错误检测机制

---

### Q2: CAN 总线的显性电平和隐性电平是什么？

**参考答案**：
- **隐性电平**：逻辑"1"，CAN_H = CAN_L = 2.5V，差分电压 0V
- **显性电平**：逻辑"0"，CAN_H = 3.5V，CAN_L = 1.5V，差分电压 2V
- **特点**：显性电平优先级高于隐性电平，会覆盖隐性电平

---

### Q3: 标准帧和扩展帧的区别是什么？

**参考答案**：
| 区别 | 标准帧 | 扩展帧 |
|------|--------|--------|
| ID 长度 | 11 位 | 29 位 |
| 仲裁场长度 | 12 位 | 32 位 |
| 新增位 | - | SRR、IDE |
| 应用场景 | 简单网络 | 复杂网络 |

---

## 8.2 仲裁与优先级类

### Q4: 什么是 CAN 总线的非破坏性仲裁？

**参考答案**：
非破坏性仲裁是指高优先级消息在仲裁过程中不会被破坏：
1. 所有节点在发送时监视总线电平
2. 发送隐性电平但检测到显性电平的节点失去仲裁权
3. 发送显性电平的节点继续发送
4. 仲裁失败的节点等待总线空闲后重试

---

### Q5: CAN 总线如何实现优先级判断？

**参考答案**：
1. ID 越小优先级越高（二进制值比较）
2. RTR 位参与仲裁：数据帧（RTR=0）优先于远程帧（RTR=1）
3. 扩展帧中 SRR 位隐性，优先级低于标准帧

---

### Q6: 位填充的作用是什么？

**参考答案**：
1. 提供同步边沿，确保接收节点能够位同步
2. 防止数据中出现与帧起始/结束标志相同的位模式
3. 错误检测：接收节点可通过检测位填充规则发现错误

---

## 8.3 错误处理类

### Q7: CAN 错误帧的作用是什么？

**参考答案**：
1. 通知总线上所有节点发生了错误
2. 其他节点接收到错误帧后丢弃当前帧
3. 发送节点会重新尝试发送
4. 确保错误不会在网络中传播

---

### Q8: 主动错误状态和被动错误状态的区别？

**参考答案**：
| 区别 | 主动错误状态 | 被动错误状态 |
|------|--------------|--------------|
| 条件 | TEC≤127 且 REC≤127 | TEC>127 或 REC>127 |
| 错误标志 | 6位显性电平 | 6位隐性电平 |
| 发送限制 | 无 | 需等待总线空闲 |

---

### Q9: 节点进入总线关闭状态后如何恢复？

**参考答案**：
1. **自动恢复**：等待 128 个总线空闲，且 TEC 清零
2. **手动恢复**：
   - 软件检测总线关闭
   - 清除错误状态
   - 重新初始化 CAN 控制器

---

## 8.4 滤波器类

### Q10: CAN 滤波器中 Filter 和 Mask 的区别？

**参考答案**：
- **Filter**：要匹配的 ID 值
- **Mask**：决定哪些位需要比较

匹配公式：`(接收ID & Mask) == (Filter & Mask)`

---

### Q11: 如何配置过滤器只接收特定范围的 ID？

**参考答案**：
例如接收 0x100-0x1FF：
```c
Filter = 0x100 << 5;
Mask = 0x700 << 5;  // 高3位为0，表示忽略
```

---

## 8.5 位时序类

### Q12: CAN 位时间由哪些段组成？

**参考答案**：
1. **同步段（SS）**：1 TQ，硬同步参考
2. **传播段（PTS）**：1-8 TQ，补偿物理延迟
3. **相位段1（PBS1）**：1-8 TQ，补偿正相位误差
4. **相位段2（PBS2）**：2-8 TQ，采样点位置

---

### Q13: 如何计算 CAN 波特率？

**参考答案**：
```
波特率 = fAPB / [(BRP + 1) × (SS + PTS + PBS1 + PBS2)]
```

例如：fAPB = 36MHz，BRP=3，各段总和 = 18 TQ
```
波特率 = 36MHz / (4 × 18) = 500 Kbps
```

---

### Q14: 采样点应该设置在什么位置？为什么？

**参考答案**：
- 位置：位时间的 75%-87.5%，通常 87.5%
- 原因：
  1. 避开信号边沿附近的噪声
  2. 给仲裁过程更多时间
  3. 确保信号稳定后采样

---

## 8.6 CANFD 类

### Q15: CANFD 与经典 CAN 的主要区别？

**参考答案**：
| 特性 | 经典 CAN | CANFD |
|------|----------|-------|
| 最高速率 | 1 Mbps | 8 Mbps |
| 数据长度 | 8 字节 | 64 字节 |
| CRC | 15 位 | 17/21 位 |
| 新增控制位 | - | FDF、BRS、ESI |

---

### Q16: CANFD 的速率切换是如何实现的？

**参考答案**：
1. 仲裁场和控制场使用较低速率（500K-1Mbps）
2. 数据场通过 BRS 位切换到高速率（2-8Mbps）
3. CRC 场和 ACK 场可继续使用高速率

---

### Q17: CANFD 如何保证与经典 CAN 的兼容性？

**参考答案**：
1. 通过 FDF 位区分 CANFD 帧和经典 CAN 帧
2. CANFD 截获模式让经典 CAN 节点通过
3. 可配置兼容模式限制特性

---

## 8.7 实际应用类

### Q18: CAN 总线常见的故障有哪些？如何排查？

**参考答案**：
**常见故障**：
1. 终端电阻缺失或过多
2. 总线短路或开路
3. 波特率配置不一致
4. 滤波器配置错误
5. 节点硬件故障

**排查方法**：
1. 使用示波器测量 CAN_H/CAN_L 电压
2. 检查终端电阻（120Ω）
3. 验证各节点波特率配置
4. 使用 CAN 分析仪抓取错误帧

---

### Q19: 如何设计 CAN 总线的 ID 分配策略？

**参考答案**：
1. 按功能模块分配 ID 范围
2. 关键系统使用高优先级（低 ID）
3. 避免 ID 冲突
4. 考虑扩展性，预留 ID 空间

---

### Q20: CAN 与 CANFD 可以共存吗？如何实现？

**参考答案**：
可以共存，但需要考虑：
1. 经典 CAN 节点无法解析 CANFD 帧
2. 使用 CANFD 截获模式
3. 或使用独立总线/网关分离

---

## 8.8 进阶问题

### Q21: CAN 总线的实时性如何保证？

**参考答案**：
1. **优先级机制**：高优先级消息优先发送
2. **非破坏性仲裁**：不因仲裁丢失数据
3. **合理的 ID 分配**：关键消息使用高优先级
4. **控制总线负载**：建议不超过 30%
5. **短数据场**：减少传输时间

---

### Q22: CAN 总线的通信延迟由哪些因素决定？

**参考答案**：
1. **仲裁延迟**：等待高优先级消息发送完成
2. **传输延迟**：数据场长度和波特率
3. **传播延迟**：总线长度和节点距离
4. **总线负载**：当前正在传输的消息
5. **重试次数**：错误导致的重发

---

### Q23: 什么情况下会选择 CANFD 而不是以太网？

**参考答案**：
| 因素 | CANFD | 以太网 |
|------|-------|--------|
| 实时性 | 优秀 | 需要特殊处理 |
| 成本 | 低 | 高 |
| 复杂度 | 低 | 高 |
| 线束 | 简单 | 复杂 |
| 确定性 | 好 | 一般 |

选择 CANFD 的场景：
- 汽车电子、工业控制
- 对实时性要求高
- 成本敏感
- 数据量不是特别大
